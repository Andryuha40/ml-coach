## Классификация как задача машинного обучения

**Классификация** — задача, в которой целевая переменная категориальна
(дискретна): нужно предсказать не число, а класс/метку объекта. Бывает:
- **бинарной** — два класса, Y = {0, 1} (или {−1, +1});
- **многоклассовой с непересекающимися классами** — объект относится ровно
  к одному из M классов, Y = {1, ..., M};
- **многоклассовой с пересекающимися классами (multi-label)** — объекту
  может быть присвоено сразу несколько меток одновременно.

Далее речь пойдёт в первую очередь о бинарной классификации — она проще
для разбора, а многоклассовая обычно сводится к комбинации бинарных задач
(например, стратегией one-vs-rest).

## Линейная модель бинарной классификации

Простейшая линейная модель для бинарной классификации (классы
закодированы как −1 и +1) предсказывает класс через знак линейной
комбинации признаков:

f(x, β) = sign(сумма βᵢ·xᵢ по всем признакам i)

- Если сумма βᵢ·xᵢ больше нуля — знак положительный, объект относится к
  положительному классу (+1).
- Если сумма меньше нуля — знак отрицательный, объект относится к
  отрицательному классу (−1).
- Уравнение «сумма βᵢ·xᵢ = 0» задаёт **разделяющую границу**: в
  двумерном случае это прямая, в многомерном — гиперплоскость.

### Обучение: минимизация доли ошибок

При обучении линейного классификатора естественно хотелось бы напрямую
минимизировать долю неверных ответов — функцию, которая для каждого
объекта равна 1, если предсказание неверное, и 0 иначе, усреднённую по
всей выборке.

### Отступ (margin)

Вводится удобное понятие — **отступ (margin)** объекта: Mᵢ = yᵢ · (β, xᵢ),
то есть произведение истинного класса объекта на значение линейной
комбинации до применения знака. Отступ показывает степень уверенности
классификатора в своём ответе:
- отступ большой положительный — объект далеко от разделяющей границы **и**
  классифицирован правильно (алгоритм «уверен» и не ошибается) — такие
  объекты называют **надёжными**;
- отступ близкий к нулю — объект рядом с границей, классификатор не очень
  уверен — **пограничные** объекты;
- отступ отрицательный — объект классифицирован неверно — **шумовые**
  объекты (или реальные выбросы/аномалии).

Задача минимизации доли ошибок эквивалентна минимизации доли объектов с
отрицательным отступом (Mᵢ < 0).

### Проблема пороговой функции потерь и гладкие аппроксимации

Такая функция потерь (доля объектов с отрицательным отступом) называется
**пороговой** — она разрывна (скачком меняется в нуле), что делает её
крайне неудобной для оптимизации градиентными методами (производная почти
везде равна нулю, а в точке разрыва не определена).

Решение — заменить пороговую функцию на гладкую **верхнюю оценку**:
непрерывную и дифференцируемую функцию, которая всегда больше или равна
пороговой по значению — тогда минимизация такой гладкой функции
автоматически «тянет вниз» и исходную пороговую. Конкретный выбор такой
функции зависит от задачи и определяет конкретный алгоритм:
- V(M) = 1 − M — кусочно-линейная (используется, например, в SVM с
  hinge loss);
- H(M) = −M — кусочно-линейная;
- L(M) = log(1 + e^(−M)) — **логистическая** (используется в логистической
  регрессии);
- Q(M) = (1 − M)² — квадратичная;
- S(M) = 2 / (1 + e^M) — сигмоидная;
- E(M) = e^(−M) — экспоненциальная (используется, в частности, в AdaBoost).

## Логистическая регрессия

### Определение и сигмоида

Логистическая регрессия — линейный классификатор, который предсказывает
не просто класс, а **вероятность** принадлежности классу: f(x, β) — это
оценка вероятности P(y = +1 | x; β). Такая вероятность моделируется через
**сигмоиду**, применённую к линейной комбинации признаков:

f(x, β) = σ(βᵀx), где σ(z) = 1 / (1 + e^(−z))

Сигмоида — это функция, которая любое вещественное число z «сжимает» в
интервал от 0 до 1, поэтому её удобно интерпретировать как вероятность.

### Почему именно сигмоида

- Область значений σ(z) — ровно отрезок [0, 1], что естественно
  соответствует определению вероятности.
- Интерпретируемая симметрия: вероятность противоположного события —
  σ(−z) = 1 − σ(z).
- У сигмоиды есть обратная функция (умение восстановить z по вероятности
  s): z(s) = ln(s / (1 − s)) — это так называемый логит, широко
  используемый в статистике.
- Сигмоида удобно дифференцируется: производная выражается через саму
  функцию — σ′(z) = σ(z)·(1 − σ(z)) — это сильно упрощает вычисление
  градиента при обучении.

### Почему не квадратичная функция потерь

Если для логистической регрессии взять обычную квадратичную функцию
потерь (сравнить предсказанную вероятность с 0/1-ответом и возвести в
квадрат), возникают две проблемы:
- получившаяся функция **не выпуклая** по параметрам β — при оптимизации
  можно застрять в локальном минимуме, не попав в глобальный;
- на совсем неправильных предсказаниях штраф получается слишком мягким:
  если модель уверенно (с вероятностью около 0%) предсказала
  положительный объект как отрицательный, штраф (1 − 0)² = 1 — то есть
  конечный и не «наказывающий» модель соразмерно грубости ошибки.

### Логистическая функция потерь (log-loss)

Вместо квадратичной используется логистическая функция потерь:

logloss = −(1/n) × сумма по всем объектам [yᵢ·log(f(xᵢ, β)) + (1 − yᵢ)·log(1 − f(xᵢ, β))]

Ключевое отличие от квадратичной: если алгоритм уверенно и правильно
предсказал класс (f → 1 при y = 1), штраф стремится к 0; но если алгоритм
уверенно и **неправильно** предсказал (f → 0 при y = 1), штраф стремится к
**плюс бесконечности** — то есть за уверенную ошибку модель наказывается
несоразмерно сильно, что и стимулирует её быть аккуратной в своих
вероятностных предсказаниях.

### Константное решение log-loss (интуиция)

Если бы модель могла выдавать только одну константу a для всех объектов
(без учёта признаков), при n₁ объектах положительного класса и n₂
объектов отрицательного класса минимум log-loss достигается ровно при
a = n₁ / n — то есть **оптимальная константа в точности равна доле
положительных объектов в выборке**. Это интуитивно понятный ориентир:
даже самая примитивная модель без признаков должна выдавать вероятность,
равную частоте класса в данных, а любая содержательная модель должна
показывать log-loss лучше этого константного бейзлайна.

## Метрики качества классификации

### Accuracy

**Accuracy** — доля правильных ответов классификатора:

accuracy = (число верно классифицированных объектов) / (общее число объектов)

**Главная проблема accuracy — дисбаланс классов.** Пример: диагностика
редкого заболевания, которое встречается у 3 человек из 1000 (0.3%).
Константный классификатор, который всегда предсказывает «здоров»,
получит accuracy = 0.997 — формально впечатляющий результат, но такая
модель совершенно бесполезна (ни разу не найдёт ни одного больного). При
сильном дисбалансе классов accuracy перестаёт отражать реальное качество
модели — необходимы другие метрики.

### Матрица ошибок (confusion matrix)

Для бинарной классификации все объекты по итогам предсказания делятся на
4 группы:

| | предсказано положительный | предсказано отрицательный |
|---|---|---|
| **истинно положительный** | TP (True Positive) | FN (False Negative) |
| **истинно отрицательный** | FP (False Positive) | TN (True Negative) |

- **Ошибка I рода (False Positive)** — истинный класс отрицательный, но
  предсказан положительный (ложная тревога).
- **Ошибка II рода (False Negative)** — истинный класс положительный, но
  предсказан отрицательный (пропуск цели).

Какая ошибка критичнее — зависит от задачи. Например, для диагностики
болезни FN (не заметить больного пациента) обычно намного опаснее, чем FP
(отправить здорового на дополнительную проверку); а для спам-фильтра,
наоборот, FP (важное письмо в спам) может быть неприемлемее, чем FN
(немного спама в почте).

### Precision (точность)

precision = TP / (TP + FP)

Показывает, **насколько можно доверять** классификатору, когда он выдал
положительный ответ: какая доля объектов, помеченных моделью как
положительные, действительно положительна.

### Recall (полнота)

recall = TP / (TP + FN)

Показывает, **как много объектов положительного класса классификатор
находит** из всех реально существующих: какая доля реально положительных
объектов была правильно найдена.

**Пример сравнения двух моделей**: модель 1 имеет precision = 0.8,
recall = 0.8; модель 2 имеет precision = 0.96, recall = 0.48. Модель 2
намного точнее в своих положительных предсказаниях, но находит намного
меньшую долю реальных положительных объектов — какая модель «лучше»,
зависит от того, что важнее в конкретной задаче: не пропустить
положительные объекты (recall) или не создавать ложных тревог
(precision).

### F1-мера

Метрика, объединяющая precision и recall в одно число — гармоническое
среднее:

F = 2 × precision × recall / (precision + recall)

Гармоническое среднее (а не обычное арифметическое) выбрано специально:
оно сильнее «штрафует» ситуацию, когда одна из двух величин низкая, даже
если другая высокая — то есть высокий F1 требует, чтобы обе метрики,
precision и recall, были достаточно хорошими одновременно.

### Порог классификации (threshold) и его регулировка

Классификатор в «мягком» варианте выдаёт не сразу класс, а некоторую
уверенность p(x) — вероятность того, что объект x относится к
положительному классу. По умолчанию, если p(x) > 0.5, объект относят к
положительному классу, иначе — к отрицательному. Но этот порог t —
не обязательно 0.5, его можно менять в диапазоне от 0 до 1 в зависимости
от задачи:
- **повышая порог** t — модель становится более «осторожной» в
  положительных предсказаниях: точность (precision) обычно растёт, а
  полнота (recall) — падает (модель находит меньше положительных
  объектов, зато почти без ложных тревог);
- **понижая порог** t — наоборот, растёт recall, но обычно падает
  precision.

Выбор конкретного порога — бизнес/предметное решение, зависящее от того,
какая из двух ошибок (FP или FN) обходится дороже в конкретной задаче.

## ROC-AUC

**ROC-AUC** — метрика, которая измеряет качество **всего семейства**
классификаторов (то есть модели при всех возможных значениях порога t)
одним числом, не привязываясь к конкретному выбранному порогу.

Дополнительные обозначения (эквивалентны части величин выше, но
традиционно используются именно в контексте ROC-кривой):
- **TPR (True Positive Rate)** = TP / (TP + FN) — то же самое, что recall
  (доля верно найденных положительных объектов);
- **FPR (False Positive Rate)** = FP / (FP + TN) — доля отрицательных
  объектов, ошибочно классифицированных как положительные.

**ROC-кривая** — кривая, состоящая из точек с координатами (FPR, TPR),
полученных при переборе всех возможных значений порога t. **AUC (Area
Under Curve)** — площадь под этой кривой, число из отрезка [0, 1]:
- AUC = 1 — идеальная классификация (существует порог, при котором TPR =
  1 и FPR = 0 одновременно — все положительные найдены, ни одной ложной
  тревоги);
- AUC = 0.5 — качество случайного классификатора (угадывающего наугад);
  на графике этому соответствует диагональная прямая из (0,0) в (1,1).

### Геометрическое построение ROC-кривой вручную

Наглядный способ построить ROC-кривую руками: упорядочить все объекты по
убыванию предсказанной классификатором вероятности/уверенности. Далее
разбить единичный квадрат на m × n частей, где m — число объектов
положительного класса, n — число объектов отрицательного класса.
Стартовать из точки (0, 0) и двигаться по списку упорядоченных объектов:
если очередной объект истинно положительный — сделать шаг **вверх** (на
1/m), если истинно отрицательный — шаг **вправо** (на 1/n). Если у
нескольких объектов подряд предсказанная вероятность совпадает, а
истинные классы разные — шаг делается «по диагонали» (одновременно вверх
и вправо, пропорционально числу объектов каждого класса среди
«связанных»). Маршрут неизбежно заканчивается в точке (1, 1) — и
получившаяся ломаная линия и есть ROC-кривая.

```python
from sklearn.metrics import roc_curve, auc

fpr, tpr, _ = roc_curve(y_true, y_score)
plt.plot(fpr, tpr, label="ROC")
print(f'Площадь по ROC-кривой составляет {auc(fpr, tpr)}')
```

### Вероятностная интерпретация AUC

Площадь под ROC-кривой равна **доле пар наблюдений** (один объект
истинно положительного класса, другой — истинно отрицательного), которые
классификатор верно упорядочил по своей предсказанной вероятности (то
есть присвоил положительному объекту в паре бо́льшую вероятность, чем
отрицательному). Это удобная и интуитивная альтернативная трактовка AUC —
как вероятности того, что случайно выбранный положительный объект
получит от модели более высокую оценку, чем случайно выбранный
отрицательный.

## AUC-PR (площадь под precision-recall кривой)

При сильном дисбалансе классов (малой доле объектов положительного
класса) ROC-AUC может давать обманчиво хороший результат — потому что FPR
считается относительно большого числа отрицательных объектов, и даже
заметное число ложных срабатываний почти не меняет FPR. Поэтому для
задач с несбалансированными классами предпочтительнее использовать
**precision-recall кривую** (кривую в координатах recall–precision при
переборе порога) и площадь под ней — **AUC-PR**:

```python
from sklearn.metrics import precision_recall_curve, average_precision_score

precision, recall, thresholds = precision_recall_curve(y_test, y_score)
pr_auc = average_precision_score(y_test, y_score)
```

## Практический код

### Классификация ирисов: базовый пример логистической регрессии

```python
from sklearn.datasets import load_iris
from sklearn.model_selection import train_test_split
from sklearn.preprocessing import StandardScaler
from sklearn.linear_model import LogisticRegression
from sklearn.metrics import f1_score

data = load_iris()
X = pd.DataFrame(data['data'], columns=data['feature_names'])
y = data['target']

# сводим многоклассовую задачу к бинарной: versicolor (класс 1) против остальных
y[y != 1] = -1

X = X[['sepal length (cm)', 'sepal width (cm)']]
X_train, X_test, y_train, y_test = train_test_split(X, y, test_size=0.3)

# ВАЖНО: fit_transform на train, transform (без повторного fit) на test — иначе утечка данных
ss = StandardScaler()
X_train = ss.fit_transform(X_train)
X_test = ss.transform(X_test)

lr = LogisticRegression()
lr.fit(X_train, y_train)
print(lr.coef_)

print('F1-мера:', f1_score(y_test, lr.predict(X_test)))
```

### Влияние коэффициента регуляризации на разделяющую границу

В sklearn у `LogisticRegression` регуляризация задаётся параметром `C`,
который является **обратным** коэффициентом регуляризации (C = 1/α):
чем меньше C, тем сильнее регуляризация.

```python
C = [0.01, 0.05, 10]
lr1 = LogisticRegression(C=C[0])
lr2 = LogisticRegression(C=C[1])
lr3 = LogisticRegression(C=C[2])
# при малом C разделяющая граница становится более простой (более сильная регуляризация,
# модель менее охотно подстраивается под конкретные точки обучающей выборки);
# при большом C граница подстраивается под данные точнее, но риск переобучения выше
```

### Подбор гиперпараметра регуляризации через кросс-валидацию

```python
from sklearn.model_selection import cross_validate

scores_lr = []
for c in np.arange(0.1, 10, 1):
    lr = LogisticRegression(C=c)
    cv_lr = cross_validate(lr, X_train, y_train, cv=5, scoring='f1')['test_score']
    scores_lr.append(cv_lr.mean())

best_c = np.arange(0.1, 10, 1)[np.argmax(scores_lr)]
lr = LogisticRegression(C=best_c)
lr.fit(X_train, y_train)
```

### Сквозной пример: EDA → предобработка → метрики (маркетинговая кампания)

Фрагменты семинара «предсказание отклика клиента на маркетинговую
кампанию» — иллюстрация полного цикла на реалистичных «грязных» данных:

```python
# обнаруживаем "мусорные" категории в данных на этапе EDA
df[(df['Marital_Status'] == 'Absurd') | (df['Marital_Status'] == 'YOLO')]
df['Marital_Status'] = np.where(
    df['Marital_Status'].isin(['Absurd', 'YOLO', 'Alone']), 'Single', df['Marital_Status']

# One-Hot Encoding категориальных признаков
data = pd.get_dummies(df, columns=['Education', 'Marital_Status'], drop_first=True)

# чистка пропусков и выбросов в конкретном признаке
data.dropna(subset=['Income'], inplace=True)
data = data[(data['Income'] <= data['Income'].quantile(0.995))]

# конструирование признаков из даты и года рождения
data['days'] = (datetime.datetime(2015, 1, 1) - pd.to_datetime(data['Dt_Customer'])).dt.days
data['Age'] = 2015 - data['Year_Birth']

# обучение и полный набор метрик, включая ROC-AUC
from sklearn.metrics import accuracy_score, recall_score, precision_score, roc_auc_score, roc_curve

lr = LogisticRegression(solver='liblinear')
lr.fit(Xtrain, ytrain)
pred_test = lr.predict(Xtest)
```

обманчиво высокую accuracy, но нулевой recall — то есть быть абсолютно
бесполезным для задачи (аналогично примеру с диагностикой редкой болезни
выше).

## Кратко: как выбирать метрику

- **Accuracy** — подходит только при примерно сбалансированных классах и
  когда ошибки обоих родов одинаково «стоят».
- **Precision** — важна, когда критична цена ложного срабатывания (FP)
  — например, если положительное предсказание запускает дорогостоящее
  действие.
- **Recall** — важна, когда критична цена пропуска положительного объекта
  (FN) — например, в медицинской диагностике или обнаружении мошенничества.
- **F1** — компромисс, когда важны и precision, и recall одновременно, а
  единого числа для сравнения моделей достаточно.
- **ROC-AUC** — хороша, когда нужно сравнить модели независимо от выбора
  конкретного порога, при относительно сбалансированных классах.
- **AUC-PR** — предпочтительнее ROC-AUC при сильном дисбалансе классов
  (подробнее про методы работы с дисбалансом — в теме `class-imbalance`).
