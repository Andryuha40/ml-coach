## Общая идея и гипотеза компактности

Метод k ближайших соседей (k Nearest Neighbors, kNN) — нелинейный
алгоритм, применимый как для классификации, так и для регрессии. Именно
с него часто начинают классические курсы машинного обучения, хотя строго
говоря это скорее эвристический метрический алгоритм, чем модель с
обучаемыми весами.

В основе метода — **гипотеза компактности**: похожие объекты находятся
близко друг к другу в пространстве признаков, а непохожие — далеко. Если
рядом с новым объектом преимущественно находятся объекты одного класса,
логично отнести к этому классу и сам объект («если объект похож на утку
и крякает как утка, то это, скорее всего, утка»).

## Алгоритм

Классификация нового объекта:
1. k — гиперпараметр алгоритма (число соседей), задаётся заранее.
2. Для нового объекта вычисляется расстояние до каждого объекта
   обучающей выборки.
3. Объекты сортируются по расстоянию, отбираются k ближайших.
4. Предсказанием становится класс, наиболее часто встречающийся среди
   этих k объектов.

Формально: предсказание — это класс y из множества всех классов Y, при
котором сумма индикаторов «i-й из k ближайших соседей принадлежит классу
y» (по всем k соседям) максимальна.

**Что происходит на этапе `fit`.** В отличие от моделей с весами
(линейная регрессия, SVM), у kNN нет процесса подбора весов по обучающей
выборке — k это гиперпараметр, задаваемый снаружи, а не обучаемый. В
«наивной» версии `fit()` просто запоминает (копирует) всю обучающую
выборку внутрь модели — формальной фазы обучения не существует. Все
вычисления (замер расстояний, сортировка, голосование) происходят уже на
этапе `predict`, при поступлении нового объекта-запроса. Это называют
**lazy learning** («ленивое обучение»).

Следствие: модель kNN весит примерно столько же, сколько сам датасет
(в отличие, например, от линейной модели, где хранится всего одна строка
параметров).

## Метрики расстояния

Второй ключевой гиперпараметр kNN — метрика, которой измеряется
расстояние между объектами.

- **Евклидова метрика**: корень из суммы квадратов разностей по
  координатам — обычное расстояние (теорема Пифагора). Наиболее частый
  выбор для непрерывных признаков.
- **Манхэттенская метрика**: сумма модулей разностей по координатам.
  Лучше работает на данных с выбросами.
- **Метрика Минковского** — обобщение евклидовой и манхэттенской метрик
  через параметр степени p (при p=1 — манхэттенская, при p=2 —
  евклидова).
- **Расстояние Хэмминга (Hamming distance)** — частный случай метрики
  Минковского для категориальных/бинарных признаков: число позиций, в
  которых значения векторов не совпадают. Пример: векторы (1,0,1,1,1) и
  (1,1,0,1,1) отличаются в 2 позициях → расстояние Хэмминга = 2.
- **Косинусное расстояние** — для текстовых данных: единица минус
  косинус угла между векторами.
- **Мера Жаккара**: отношение мощности пересечения множеств к мощности
  их объединения — часто применяется для измерения похожести текстов
  (подробнее см. раздел про LSH ниже).

**Пример вычисления.** Даны два вектора (1, 1, 2) и (2, 0, 3).
Манхэттенское расстояние: |1−2| + |1−0| + |2−3| = 1 + 1 + 1 = 3.

## Масштабирование признаков

Масштабировать (нормализовать) данные перед применением kNN нужно
обязательно. Причина: kNN считает расстояние сразу по всем признакам, и
если признаки имеют разный масштаб, признак с большим разбросом значений
«забивает» остальные.

Пример из семинара: два мужчины 31 и 38 лет с очень разным доходом
(порядка миллиона у одного и порядка 120 условных единиц у другого) и
женщина существенно старше, с доходом того же порядка, что и у первого
мужчины. По возрасту и полу оба мужчины гораздо ближе друг к другу, чем к
женщине, но при подсчёте евклидова расстояния по немасштабированным
признакам огромная разница в доходе «задавит» вклад возраста, и окажется,
что мужчина с высоким доходом ближе по расстоянию к женщине с высоким
доходом, чем к другому мужчине с низким доходом — хотя по существу они
должны быть похожи скорее по возрасту и полу.

Важное уточнение: если k уже зафиксировано (модель «обучена», то есть
выборка сохранена), простое масштабирование данных задним числом не
меняет порядок близости объектов друг к другу — только сжимает или
расширяет расстояния пропорционально. Поэтому масштабирование нужно
закладывать заранее, до использования модели, а не как способ
«постфактум» исправить неудачное предсказание.

## Взвешенный (обобщённый) kNN

В «наивном» алгоритме все k ближайших соседей учитываются с одинаковым
весом. Из-за этого возможна контринтуитивная ситуация: среди k ближайших
соседей нового объекта оказывается больше представителей более далёкого
класса, чем ближнего, и алгоритм выдаёт неожиданный ответ (например,
объект явно рядом с красным кружком, но алгоритм предсказывает зелёный
треугольник, потому что зелёных треугольников среди k ближайших оказалось
больше).

Идея исправления: чем ближе сосед к объекту-запросу, тем больше должен
быть его вес при голосовании.

**Взвешивание по порядку (рангу).** Соседей упорядочивают по возрастанию
расстояния и присваивают вес, обратно пропорциональный порядковому
номеру: ближайшему — вес 1, следующему — 1/2, следующему — 1/3 и так
далее.

**Метод парзеновского окна.** Более продвинутый вариант — учитывать не
только порядок, но и само значение расстояния через функцию-ядро K,
которая ставит в соответствие расстоянию некоторый коэффициент (вес).
Предсказание — класс y, максимизирующий сумму по k соседям от
[индикатор принадлежности классу y] × K(расстояние до соседа / h), где h
— ширина окна. Примеры ядер:

- **Прямоугольное**: 1/2 при |x| ≤ 1, иначе 0 — объекты дальше ширины окна
  h получают нулевой вес и не влияют на предсказание.
- **Треугольное** — ещё один вариант функции ядра.
- **Гауссовское**: 1/корень(2π) × экспонента от минус x²/2.

Запоминать конкретные формулы ядер не обязательно — важен общий принцип:
чем дальше объект от запроса, тем меньше его вклад в голосование, а
очень далёкие объекты могут обнуляться совсем. Обратная сторона: если
рядом с запросом случайно оказывается аномальный (выбросовый) сосед,
из-за высокого веса, придаваемого близким объектам, он может сильнее
повлиять на предсказание — учёт расстояния немного повышает риск
переобучения (излишней чувствительности к выбросам).

## kNN для регрессии

Наивный подход: предсказанием становится среднее значение целевой
переменной среди k ближайших соседей (вместо голосования по классам, как
в классификации).

Взвешенный вариант — **формула Надарая-Ватсона** (взвешенное среднее с
учётом расстояния до соседей, по аналогии с методом парзеновского окна):
предсказание равно сумме по всем соседям от (ответ соседа × вес по ядру
от расстояния до соседа), делённой на сумму весов по ядру. Похожий
принцип используется, например, при оценке стоимости жилья: часто
ориентируются на цены похожих, ближайших по характеристикам объектов.

## Гиперпараметр k: переобучение и недообучение

Формализация из ноутбука: пусть k — число соседей; для каждого объекта u
берутся k ближайших к нему объектов из тренировочной выборки, и класс
объекта u определяется как argmax по y от суммы индикаторов «сосед
принадлежит классу y» (для взвешенного варианта — с весом, обратно
пропорциональным расстоянию до соседа).

С ростом k модель становится более устойчивой к случайным выбросам и
шуму в данных, но при слишком большом k модель чрезмерно упрощается и
перестаёт улавливать реальные закономерности — наступает недообучение
(в пределе, при k равном числу объектов в выборке, для всех запросов
будет предсказываться один и тот же ответ). При маленьком k, наоборот,
велика вероятность реагировать на шумовой объект-выброс — модель
переобучается (при k=1 вокруг каждого шумового объекта образуется
«остров» предсказания не того класса, что логически некорректно).

Таким образом увеличение k — способ борьбы с переобучением, но при
чрезмерном увеличении k алгоритм скатывается в недообучение. Оптимальное
значение k нужно подбирать по валидационной выборке или кросс-валидацией
(между этими крайностями) — сравнивать разные k по качеству на train
бесполезно: каждый объект является ближайшим сам к себе, и оптимальным
всегда окажется k=1.

## Преимущества и недостатки kNN

Преимущества: умеет решать и классификацию, и регрессию; не делает
предположений о линейной разделимости или распределении данных; неплохо
интерпретируется (можно объяснить ответ через ближайших соседей); не
требует формального обучения (lazy learning).

Недостатки:
1. Модель хранит всю обучающую выборку целиком — требует много памяти.
2. Для каждого предсказания нужно вычислить расстояние от запроса до
   всех объектов выборки — дорого по вычислениям.
3. Данные обязательно нужно масштабировать.
4. Нужно подбирать число соседей k и метрику расстояния, которая
   действительно хорошо отражает похожесть объектов в конкретной задаче —
   не всегда тривиально.

## Вычислительная сложность kNN

Чтобы найти k ближайших соседей для одного объекта-запроса, нужно
вычислить расстояние до каждого из l объектов обучающей выборки; вычисление
расстояния между парой объектов занимает O(d) операций, где d — число
признаков. Итоговая сложность одного предсказания — **O(l · d)**.

Задача поиска ближайших соседей актуальна не только в kNN, но и,
например, при поиске похожих текстов или в векторных базах данных. Для
больших объёмов данных точный перебор становится слишком дорогим —
используются алгоритмы **приближённого** поиска ближайших соседей,
которые находят соседей быстрее, сохраняя приемлемую точность.

## LSH (Locality-Sensitive Hashing) — приближённый поиск соседей

Общая идея: объекты пропускаются через хеш-функцию, которая распределяет
их по «корзинам» (бакетам). При удачно подобранной хеш-функции похожие
объекты чаще попадают в одинаковые (или соседние) корзины при разных
случайных хешированиях, а непохожие — почти всегда в разные. Тогда для
поиска соседей нового объекта достаточно посмотреть только объекты из
той же корзины, а не всю выборку.

### Метод случайных проекций

Подходит для табличных данных, где объект представлен точкой в
многомерном пространстве признаков.

1. Через начало координат проводится несколько случайных гиперплоскостей.
2. Для каждой гиперплоскости считается скалярное произведение объекта с
   вектором, задающим гиперплоскость: если оно положительное — ставится
   1, если отрицательное — 0.
3. После нескольких таких гиперплоскостей (например, 10) каждый объект
   кодируется битовой последовательностью.
4. Похожесть двух объектов оценивается через расстояние Хэмминга между
   их битовыми кодами. В разобранном на занятии примере с 6
   гиперплоскостями два объекта совпали по 5 из 6 позиций (похожесть —
   5/6).

### MinHash

Используется, в частности, для оценки похожести текстов.

**Мера Жаккара.** Для множеств A и B: отношение мощности пересечения к
мощности объединения. Текст (документ) разбивается на n-граммы (шинглы) —
например, биграммы. Слово «Саша» → биграммы {са, аш, ша}, слово «Маша» →
{ма, аш, ша}. Пересечение — {аш, ша} (2 элемента), объединение — {ма, аш,
ша, са} (4 элемента). Мера Жаккара = 2/4 = 0.5.

**Матрица «шинглы — документы».** По строкам — все уникальные шинглы
корпуса, по столбцам — документы; на пересечении — 1, если шингл
встречается в документе, иначе 0. На практике такая матрица огромна
(десятки тысяч шинглов на миллионы документов) и разрежена (в основном
нули) — хранить и обрабатывать её напрямую дорого.

**Алгоритм построения сигнатур MinHash** — сжимает эту большую разреженную
матрицу в компактную матрицу сигнатур, приближённо сохраняющую меру
Жаккара:

1. Все шинглы (строки матрицы) нумеруются.
2. Строки матрицы случайно перемешиваются (случайная перестановка
   индексов строк).
3. Для каждого документа (столбца), просматривая строки сверху вниз в
   перемешанном порядке, находится минимальный индекс строки, где стоит
   1 — это значение записывается как хеш документа для данной
   перестановки.
4. Процедура (новая случайная перестановка → новый минимальный индекс)
   повторяется много раз (число повторов — гиперпараметр алгоритма: чем
   больше перестановок, тем точнее приближение к истинной мере Жаккара).
5. Результат — компактная матрица сигнатур: число строк равно числу
   перестановок, число столбцов — числу документов.

Мера Жаккара, посчитанная по матрице сигнатур, приближается к мере
Жаккара, посчитанной по исходной (большой, разреженной) матрице шинглов.

### Поиск похожих документов: разбиение на бэнды (banding)

Финальный шаг — поиск похожих документов с некоторой заданной
вероятностью:

1. Матрица сигнатур разбивается на «бэнды» (полосы) равной ширины —
   группы строк с одинаковым числом строк в каждой (например, матрица из
   100 перестановок делится на 25 бэндов по 4 строки).
2. Внутри каждого бэнда столбцы (куски сигнатур документов) хешируются в
   отдельные бакеты.
3. Два документа считаются похожими, если их столбцы попали в один и тот
   же бакет хотя бы в одном бэнде (не обязательно во всех).

Если s — оценка похожести (мера Жаккара) по одной строке сигнатуры, а r —
число строк в одном бэнде, то вероятность совпадения в конкретном бэнде —
s в степени r, а вероятность не совпасть ни в одном из b бэндов — (1 − s
в степени r) в степени b. При достаточном числе бэндов эта вероятность
становится очень маленькой — то есть с высокой вероятностью похожие
документы будут пойманы хотя бы в одном бэнде.

LSH на практике активно применяется в задачах поиска ближайших соседей в
больших объёмах данных, в частности в векторных базах данных.

## Дополнительно: разложение ошибки на смещение и разброс (bias-variance)

Этот материал разобран на том же семинаре (13), сразу после LSH — как
общий инструмент для оценки качества любой модели, не только kNN.

**Вероятностная модель данных.** Предполагается, что целевая переменная y
описывается истинной (неизвестной) зависимостью f(x) плюс шум: y = f(x) +
ε, где шум ε распределён нормально со средним 0.

**Разложение ошибки.** Ошибка модели a(x) на фиксированном объекте —
математическое ожидание квадрата разности между истинным ответом и
предсказанием модели. Раскладывая это выражение, получаем сумму трёх
слагаемых: квадрат смещения (Bias²) + разброс (Variance) + шум (σ²).

- **Смещение (Bias)** — насколько в среднем предсказание модели
  отличается от истинного значения; отражает, насколько хорошо модель в
  принципе способна выучить целевую зависимость.
- **Разброс (Variance)** — насколько сильно могут различаться
  предсказания модели, если обучать её на разных обучающих подвыборках;
  отражает переобученность модели.
- **Шум (σ², неустранимая ошибка)** — связан со случайностью в самих
  данных; на эту компоненту модель повлиять не может.

**Иллюстрация «мишень для дартса»**: низкое смещение + низкий разброс —
предсказания точные и кучные (близко к идеалу); низкое смещение +
высокий разброс — в среднем точно, но конкретные предсказания сильно
разбросаны (риск переобучения); высокое смещение + низкий разброс —
предсказания стабильно кучные, но систематически далеки от истины;
высокое смещение + высокий разброс — худший случай.

**Влияние сложности модели.** При усложнении модели (рост числа
параметров) смещение обычно уменьшается (модель точнее подстраивается
под обучающую выборку), но одновременно растёт разброс (модель сильнее
подстраивается под конкретные особенности обучающей выборки —
переобучение). Задача — найти оптимальную сложность модели, при которой
суммарная ошибка (Bias² + Variance) минимальна. Эта закономерность — не
строгий закон, а эмпирическое правило (rule of thumb).

## Дополнительно: наивный байесовский классификатор (кратко)

Наивный Байес (Naïve Bayes) — алгоритм обучения с учителем, основанный на
теореме Байеса с «наивным» предположением об условной независимости
между каждой парой признаков при заданном значении класса. Предсказание
— класс с максимальной апостериорной вероятностью (знаменатель теоремы
Байеса — полная вероятность признаков — можно не вычислять, так как он
одинаков для всех классов). Благодаря предположению о независимости,
совместная вероятность признаков при данном классе раскладывается в
произведение отдельных условных вероятностей каждого признака.

Часто используется **гауссов наивный Байес** — приближает числовые
признаки нормальным распределением, параметры которого (среднее и
дисперсия по каждому классу) оцениваются методом максимального
правдоподобия. Другие варианты: многочленный (для текста, например,
спам-фильтров), Бернулли (бинарные признаки), категориальный.

Плюсы: простота реализации, низкие вычислительные затраты, оптимален,
если признаки действительно независимы. Минус: в большинстве реальных
задач признаки не полностью независимы, поэтому качество часто страдает.

## Код: разобранные задачи и практика на датасете digits

### Задача (расчёт вручную)

Классификация на 3 класса по 2 признакам, k=3, манхэттенская метрика,
обучающая выборка:

| Признак 1 | Признак 2 | Класс |
|-----------|-----------|-------|
| 1         | -1        | 1     |
| 2         | 2         | 1     |
| 3         | 2         | 2     |
| 1         | 0         | 3     |
| 2         | -2        | 3     |

Предсказание для объекта x=(2, -1): расстояния до объектов 1-5 (манхэттен)
— 1, 3, 4, 2, 1. Три ближайших — объекты 1, 4, 5 (классы 1, 3, 3).
Большинство — класс 3, предсказание: 3.

Та же выборка признаков, но с непрерывными ответами (регрессия), k=3:
соседи те же (1, 4, 5) с ответами 3.5, -0.4, 0.1 — предсказание: среднее
(3.5 − 0.4 + 0.1) / 3 = 1.1.

### Практический код (датасет digits, подбор k по валидационной выборке)

```python
import numpy as np
import pandas as pd
import matplotlib.pyplot as plt
import sklearn

from sklearn.datasets import load_digits
from sklearn.neighbors import KNeighborsClassifier
from sklearn.utils import shuffle

clf = KNeighborsClassifier(n_neighbors=3)

data = load_digits()
X = data.images
y = data.target

# вытягиваем квадратное изображение в вектор, чтобы получить матрицу объекты-признаки
X = X.reshape(X.shape[0], -1)

# перемешиваем данные
X, y = shuffle(X, y)

X_train, y_train = X[:700, :], y[:700]
X_val, y_val = X[700:1300, :], y[700:1300]
X_test, y_test = X[1300:, :], y[1300:]

# Обучаем классификатор и делаем предсказания
clf.fit(X_train, y_train)
y_predicted = clf.predict(X_test)

# Вычисляем простейшую метрику качества алгоритма - долю правильных ответов
print("Accuracy is: ", np.mean(y_test==y_predicted))
```

```python
# Подбор k на валидационной выборке:
k_best = -1
best_accuracy = 0

for k in range(1, 20):
    y_predicted = KNeighborsClassifier(n_neighbors=k).fit(X_train, y_train).predict(X_val)

    val_accuracy = np.mean(y_predicted==y_val)
    print(f"k = {k}; accuracy = {val_accuracy:.3f}")

    if val_accuracy > best_accuracy:
        best_accuracy = val_accuracy
        k_best = k

k_best
```

```python
clf = KNeighborsClassifier(n_neighbors=k_best)
clf.fit(X_train, y_train)

for X_data, y_data in zip([X_train, X_val, X_test], [y_train, y_val, y_test]):
    y_predicted = clf.predict(X_data)
    print(f"Accuracy: {np.mean(y_predicted==y_data):.3f}")
```

Вывод из ноутбука: качество на обучающей выборке самое лучшее, но оно
обманчиво (алгоритм уже знает эти объекты); качество на валидационной
выборке тоже не совсем честное (использовалось для подбора k); честная
оценка — только качество на тестовой выборке, которая нигде не
использовалась ни в обучении, ни в подборе гиперпараметров.
