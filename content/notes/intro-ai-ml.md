## Коротко из истории ИИ

- **1947** — Алан Тьюринг высказывает идею «интеллектуальных машин»,
  меняющих внутреннее состояние на основе опыта; в 1950 предлагает тест
  Тьюринга для оценки интеллекта компьютера.
- **1956** — на Дартмутской конференции Джон Маккарти вводит термин
  «искусственный интеллект».
- **1960** — перцептрон Розенблатта — первый рабочий алгоритм,
  моделирующий работу нейрона мозга.
- **1966** — диалоговая система Элиза имитирует разговор с психотерапевтом,
  подставляя значимые слова пациента в шаблонные фразы (одна из первых
  систем обработки естественного языка).
- **1970-1980-е** — расцвет экспертных систем на основе правил (например,
  MYCIN для диагностики бактериальных инфекций); в 1984 — первый алгоритм
  автоматического построения решающего дерева.
- **1997** — Deep Blue (IBM) обыгрывает Гарри Каспарова в шахматы.
- **2002** — Roomba, первый бытовой робот-пылесос.
- **2016** — нейросетевой машинный перевод существенно обгоняет по качеству
  предыдущие подходы на правилах.
- **2017** — статья *Attention Is All You Need* вводит механизм внимания,
  который лёг в основу почти всех современных SOTA-решений.
- **2022-2023** — взрывной рост генеративных моделей и больших языковых
  моделей (LLM), например ChatGPT.

Три волны, которые стоит различать по времени появления: машинное обучение
(с начала 1990-х) → глубинное обучение / нейросети (с середины 2000-х) →
генеративный ИИ и LLM (с 2022 года).

## Что такое машинное обучение

Три равноценных определения, которые встречаются в курсе:
- наука об извлечении закономерностей из ограниченного числа примеров
  (Е. Соколов);
- наука об алгоритмах, автоматически улучшающихся благодаря опыту
  (Яндекс);
- процесс, дающий компьютерам возможность обучаться выполнять что-то без
  явного написания кода под конкретную задачу (А. Самуэль).

Общая идея всех трёх: вместо того чтобы вручную прописывать правила
(`если... то...`), мы показываем алгоритму примеры и он сам находит
закономерность.

## Объект, признаки, обучающая выборка

Чтобы применить алгоритм ML к чему угодно (машине, тексту, пациенту), это
что-то сначала нужно **формализовать** — представить набором чисел,
который называется **признаковым описанием**.

- **Объект** — сущность, для которой хотим сделать предсказание (клиент,
  квартира, фотография...).
- **Признак** — одна координата признакового описания, конкретная
  измеримая характеристика объекта.
- **Обучающая выборка** — набор объектов, для которых уже известен
  правильный ответ.
- **Целевая переменная (ответ)** — то, что мы хотим предсказать.
- **Модель (алгоритм)** — функция, которая по признаковому описанию
  объекта выдаёт предсказание.

Пример — задача скоринга: по характеристикам клиента (пол, возраст, доход,
кредитная история...) предсказать, вернёт ли он кредит. Тут: объекты —
клиенты из датасета, признаки — их характеристики, целевая переменная —
число 1 (вернёт) или 0 (не вернёт).

Пример признакового описания вручную: возьмём автомобиль. Его можно описать
цветом, мощностью двигателя, наличием аудиосистемы, максимальной скоростью,
массой, маркой. Нечисловые характеристики (цвет, марка) переводятся в числа
через нумерацию возможных значений: например, цвета «зелёный/красный/синий»
кодируются как 0/1/2, наличие аудиосистемы — как 0/1. Тогда конкретный
автомобиль превращается в вектор чисел вида [1, 100, 1, 150, 800, 0] —
именно такой вектор и подаётся в алгоритм.

### Типы признаков

- **Бинарные** — ровно два значения (обычно 0/1 или -1/1). Пример: наличие
  аудиосистемы, пол, «купил / не купил».
- **Категориальные** — конечное множество значений, для модели их нужно
  закодировать числами. Делятся на:
  - **номинальные** — значения нельзя сравнивать («синий цвет больше
    красного» бессмысленно) — например, марка автомобиля, цвет;
  - **ранговые** — значения упорядочены («майор старше сержанта» имеет
    смысл) — например, воинское звание, оценка на экзамене.
- **Непрерывные** — число из непрерывного диапазона (масса, цена, рост).
  Могут быть целыми или вещественными.
- **Географические** — пары широта/долгота, требуют специфической
  обработки (задачи логистики).
- **Дата и время** — специальный формат (`datetime` в pandas), важен для
  временных рядов и индексирования по времени.

## Модель как семейство функций

Наша цель — найти алгоритм, который очень похож на скрытую от нас
закономерность (истинную зависимость между признаками и ответом). Мы сами
решаем, из какого класса функций выбирать этот алгоритм — это и называется
**выбором модели**.

Модель — это не одна конкретная функция, а целое **семейство функций**,
зависящих от параметров. Обучение — это подбор конкретных значений
параметров внутри этого семейства.

Пример: пусть мы подозреваем, что истинная зависимость похожа на параболу.
Тогда мы выбираем модель — семейство всех многочленов второй степени вида
`A(x) = a·x² + b·x + c`, где a, b, c — параметры. Обучение — это подбор
конкретных a, b, c, при которых эта парабола лучше всего описывает данные.
Когда параметры подобраны — получаем конкретную функцию, которая и есть наш
итоговый алгоритм.

## Функция потерь и задача ML как задача оптимизации

Формальные обозначения курса (пригодятся и в других темах):
- N — число объектов (строк) в выборке; M — число признаков (столбцов).
- x — один объект (точнее, его признаковое описание, вектор чисел);
  X — весь набор объектов (матрица).
- y — правильный ответ для объекта (или вектор ответов для набора объектов).
- a(x) — результат применения алгоритма a к объекту x, то есть предсказание.

**Функция потерь** — функция, которая по объекту x и правильному ответу y
показывает, насколько сильно ошибся алгоритм на этом одном объекте. Часто
её обозначают L(x, y) (более полная запись — L при индексе алгоритма a).
Например, для задачи регрессии (предсказание стоимости автомобиля по его
признакам) функцией потерь может быть просто модуль разницы между
предсказанием и истинным значением: «насколько по модулю предсказанная
цена отличается от настоящей».

Дальше рассуждение такое: мы хотим не просто маленькую ошибку на
конкретном объекте, а маленькую ошибку **в среднем по всей обучающей
выборке** — то есть чтобы модель хорошо работала не на одном примере, а
вообще. Поэтому задача формулируется так: посчитать среднее значение
функции потерь по всем N объектам обучающей выборки и подобрать параметры
модели так, чтобы это среднее было минимально возможным. Это и называют
**эмпирическим риском**, а его минимизация по параметрам — то же самое, что
обучение модели.

Получается ключевая идея курса: **задача машинного обучения сводится к
задаче оптимизации** — минимизации средней функции потерь по параметрам
модели. Решение самой задачи оптимизации (как именно искать минимум) —
отдельная область (см. тему про градиентный спуск).

## Типы задач машинного обучения

- **Регрессия** — целевая переменная непрерывна (число). Примеры:
  стоимость квартиры, прибыль ресторана, зарплата выпускника.
  Общий вид линейной модели: предсказание = свободный член + сумма
  (коэффициент × признак) по всем признакам, плюс ошибка. Коэффициенты —
  это и есть параметры модели, которые подбираются при обучении.
- **Классификация** — целевая переменная категориальна (класс/метка).
  Бывает бинарной (два класса, например «болен/здоров»), многоклассовой
  (класс — один из M вариантов, например порода животного) или
  многоклассовой с пересекающимися классами (объект может относиться сразу
  к нескольким меткам). Примеры: медицинская диагностика, кредитный
  скоринг, отток клиентов, клик по баннеру.
- **Кластеризация** — обучение без учителя: нужно разбить объекты на
  группы (кластеры) так, чтобы объекты внутри группы были похожи друг на
  друга, а из разных групп — различны. В отличие от классификации, здесь
  **нет** заранее известных правильных ответов и обучающей выборки в
  привычном смысле — классы не заданы заранее, алгоритм сам находит
  структуру в данных.
- Другие типы задач (кратко, без деталей на этом уровне курса):
  ранжирование, рекомендации, снижение размерности, поиск аномалий,
  прогнозирование временных рядов, детекция объектов.

## Процесс решения ML-задачи и разведочный анализ данных (EDA) — коротко

Типичный пайплайн: получение данных → разведочный анализ (EDA) → отбор/
конструирование признаков → выбор модели и обучение на тренировочной
выборке → оценка качества на тестовой выборке → внедрение в продакшен.

EDA (Exploratory Data Analysis) — это первый содержательный шаг после
получения данных: понять объём и полноту данных, характеристики признаков,
их распределения, зависимости между признаками и с целевой переменной,
найти пропуски и выбросы. Подробный разбор техник EDA и `sklearn.Pipeline`
— в теме «Разведочный анализ данных, sklearn Pipeline, кодирование
признаков» (`eda-and-pipeline`); здесь важно только само место EDA в общем
процессе.

## Обучение и валидация: train/test split

Разбиение данных на **обучающую (train)** и **тестовую (test)** выборки —
одна из центральных идей курса. Смысл: если оценивать качество модели на
тех же данных, на которых она обучалась, оценка будет обманчиво
оптимистичной — модель могла просто подстроиться под эти конкретные
примеры, а не выучить общую закономерность.

Аналогия курса: это как домашняя работа и контрольная в школе. Train —
это домашнее задание: оно тренирует, и его более-менее может решить и
слабый, и сильный ученик (в том числе подсмотрев ответы). Test — это
контрольная: задачи, которых ученик раньше не видел. Контрольную хорошо
решит только тот, кто реально разобрался в теме, а не запомнил конкретные
примеры. Поэтому качество модели корректно оценивать только на test.

Иногда выборку делят не на две, а на три части — добавляется
**валидационная** выборка. Она нужна для подбора гиперпараметров модели
(и для отслеживания качества по ходу обучения у моделей, которые учатся
долго, например нейросетей) — то есть валидационная выборка выполняет роль
«домашки» именно для гиперпараметров, а итоговую проверку всё равно делают
на отдельном test. В курсе такое трёхчастное разбиение почти не
встречается за пределами нейросетей.

### Параметры vs гиперпараметры

- **Параметры** модели подбираются автоматически в процессе обучения (на
  train). Пример: веса в линейной регрессии, структура решающего дерева.
- **Гиперпараметры** задаются и фиксируются заранее, до обучения, вручную
  или через отдельный подбор. Пример: learning rate градиентного спуска,
  сила регуляризации, глубина дерева.

## Интерфейс sklearn (на примере Ирисов Фишера)

Ирисы Фишера — классический учебный датасет: для каждого из 150 цветков
трёх видов измерены 4 непрерывных признака (длина/ширина чашелистика и
лепестка), задача — классификация по виду.

Библиотека `sklearn` задаёт единый интерфейс почти для всех алгоритмов
классического ML — умение это использовать переносится на линейные модели,
деревья, SVM, ансамбли, kNN и т.д. (модули `sklearn.linear_model`,
`sklearn.tree`, `sklearn.svm`, `sklearn.ensemble`, `sklearn.cluster`,
`sklearn.neighbors`).

Базовый рабочий процесс — код ниже воспроизводится дословно из семинара
(эталон для проверки код-упражнений по этой теме):

```python
import pandas as pd
import seaborn as sns

import sklearn

from sklearn.datasets import load_iris
from sklearn.metrics import accuracy_score
from sklearn.linear_model import LogisticRegression
from sklearn.model_selection import train_test_split

# Загрузка датасета
X, y = load_iris(return_X_y=True)

# Разбиение выборки на тренировочную и тестовую
X_train, x_test, y_train, y_test = train_test_split(X, y, train_size=0.8, shuffle=True)

# Обучение модели
model = LogisticRegression().fit(X_train, y_train)

# Предсказание на тестовой выборке
preds = model.predict(x_test)

# Оценка качества — доля верных ответов
acc = accuracy_score(y_test, preds)
```

Три ключевых метода, общих почти для всех моделей sklearn:
- `model.fit(X, y)` — обучение (подбор параметров) на признаках `X` и
  ответах `y`; для обучения без учителя — `model.fit(X)` без `y`.
- `model.predict(X)` — получить предсказания модели для новых объектов
  (модель их не видела при обучении).
- `accuracy_score(y_true, y_pred)` из `sklearn.metrics` — доля верных
  ответов: одна из простейших метрик качества классификации (подробнее про
  метрики — в теме `classification-metrics`).

## Мини-глоссарий обозначений (для справки)

- N — число объектов, M — число признаков.
- x, X — объект (вектор признаков) и матрица объектов соответственно.
- y — правильный ответ (или вектор ответов).
- a(x) — предсказание алгоритма a на объекте x.
- L(x, y) — функция потерь: величина ошибки алгоритма на объекте x при
  правильном ответе y.
- «Эмпирический риск» — среднее значение функции потерь по всей обучающей
  выборке; обучение = минимизация эмпирического риска по параметрам модели.
