## Зачем нужны отбор признаков и понижение размерности

Отбор признаков — процедура, в результате которой из данных удаляются
некоторые признаки. Причины:

- Слишком много признаков — плохо для моделей классического машинного
  обучения: они, в отличие от глубоких моделей, не умеют эффективно
  учиться на избыточно большом количестве признаков, и это увеличивает
  время обучения.
- Лишние признаки, не связанные с целевой переменной, создают шум,
  усложняют модели поиск реальной зависимости и способствуют
  переобучению.
- Признаки, сильно зависимые (коррелирующие) друг с другом
  (мультиколлинеарность), тоже стоит частично убирать, чтобы модель не
  переобучалась.

### Проклятие размерности

Чем выше размерность данных (больше признаков), тем:
- более разреженными становятся данные и сложнее вычисления;
- сильнее растёт риск переобучения;
- менее информативным становится расстояние между точками (что особенно
  критично для метрических методов вроде kNN — см. тему `knn`);
- экспоненциально больше данных требуется для сохранения того же
  качества модели.

### Feature selection vs feature extraction

Два принципиально разных способа понижения размерности:

| | Отбор признаков (Feature Selection) | Извлечение признаков (Feature Extraction) |
|---|---|---|
| Подход | Выбирает подмножество релевантных признаков из исходного набора | Преобразует исходные признаки в новый, более информативный набор |
| Механизм | Снижает размерность, сохраняя исходные признаки | Снижает размерность за счёт преобразования данных в новое пространство |
| Требования | Требует предметных знаний и инженерии признаков | Может применяться к необработанным данным без предварительной инженерии |
| Минусы | Может привести к потере полезной информации при удалении важных признаков | Может внести избыточность и шум, если извлечённые признаки определены плохо |
| Примеры методов | Filter, Wrapper, Embedded | PCA, LDA, Kernel PCA, автоэнкодеры |

## Отбор признаков: фильтрационные методы (Filter)

Признаки отбираются на основе заранее посчитанной статистики — самые
быстрые методы, но не учитывают совокупное (комбинированное) влияние
признаков друг на друга.

### Квазипостоянные признаки (порог по дисперсии, Variance Threshold)

Если признак почти на всех объектах принимает одно и то же (или почти
одно и то же) значение — у него маленькая дисперсия, и его можно считать
почти константой, не несущей значимой информации, и удалить. Точного
универсального порога нет (эмпирический ориентир — условно, дисперсия
меньше единицы). Надёжный способ проверки: удалить подозрительный
признак и посмотреть, изменилось ли качество модели — если нет, признак
действительно можно убирать.

### Отбор по корреляции с целевой переменной

Для каждого признака считается его корреляция с таргетом, убираются
признаки с маленькой по модулю корреляцией. Плюс — скорость (корреляция
считается один раз для каждого признака). Основные коэффициенты:
Пирсона (классический, максимум 1, отсутствие связи — 0), Спирмена,
Кендалла.

Недостатки:
- отсутствие (линейной, пирсоновской) корреляции не означает отсутствие
  связи — может быть сложная нелинейная зависимость; для её поимки есть
  другие коэффициенты корреляции;
- корреляция не означает причинно-следственную связь (классический
  иллюстративный пример — псевдокорреляция между расходами США на науку
  и космос и числом самоубийств через удушение, коэффициент около 0.87 —
  реальной причинной связи здесь, скорее всего, нет);
- **главная проблема всех одномерных методов** — они не учитывают
  совокупное влияние признаков. Пример с фруктами: форма сама по себе
  слабо связана с ответом «яблоко ли это», цвет тоже слабо коррелирует
  сам по себе — но сочетание «красный и круглый» уже достаточно
  однозначно указывает на яблоко.

### Ограничения одномерных (univariate) методов — подробнее

Пример из семинара: пусть в двумерном пространстве признаков (x1, x2)
классы линейно разделимы. Если спроецировать выборку на одну ось (то есть
отбросить второй признак), объекты классов частично перемешаются, и
разделимость исчезнет. При этом одномерная оценка важности может
показать, что каждый признак по отдельности слабо информативен, и
предложить удалить оба — хотя вместе они дают хорошую разделимость. Ещё
сильнее проблема проявляется, если признаки влияют на таргет только в
комбинации (взаимодействии). Правило: «вместе с водой не выплеснуть
ребёнка» — методы отбора признаков могут как улучшить модель, так и не
дать эффекта, а иногда и ухудшить качество.

### Критерий хи-квадрат (χ²)

Проверяет, насколько наблюдаемые частоты соответствуют теоретически
ожидаемым в предположении о независимости переменных. Формула: сумма по
всем ячейкам от (наблюдаемая частота − ожидаемая частота)² / ожидаемая
частота. Чем больше значение χ², тем сильнее взаимосвязь между
переменными. Работает для категориальных признаков — аналог коэффициента
корреляции, но для категориальных данных. После расчёта χ² для всех
признаков задаётся порог отсечения, либо признаки сортируются по
убыванию χ² и наименее значимые отбрасываются.

### Взаимная информация (mutual information)

Оценивает силу связи между признаком и таргетом на основе совместной
плотности вероятности двух переменных, делённой на произведение их
отдельных (маргинальных) плотностей — показывает, сколько информации о
целевой переменной содержится в конкретном признаке. Признаки с маленьким
значением взаимной информации считаются малополезными. Реализовано «из
коробки» в scikit-learn.

Похожим образом признак можно оценивать через **ROC-AUC**: рассматривая
сам признак как предсказание таргета и считая AUC — чем выше AUC, тем
важнее признак.

### Матрица Phik (φk)

Общепринятый подход, опирающийся под капотом на критерий хи-квадрат и
другие статистики — позволяет единообразно оценить связь между числовыми
и категориальными признаками сразу для всей таблицы («все со всеми»).

**Важная оговорка про все фильтрационные методы**: с точки зрения строгой
статистики их применение для отбора признаков в машинном обучении — не
совсем строгая, «вольная» трактовка. Тем не менее на практике такой
подход хорошо работает, даже если теория и практика здесь не полностью
сходятся. Также стоит быть готовым, что случайный «шумовой» признак
иногда неожиданно оказывается в верхней части рейтинга важности по любому
из этих методов — это нормальная особенность подобных экспериментов, а
не повод считать метод сломанным.

## Оберточные методы (Wrapper methods)

Используют жадный (последовательный) отбор признаков — самые точные с
практической точки зрения, но заметно более медленные, чем
фильтрационные.

**Backward elimination (последовательное исключение).** Пусть есть 1000
признаков, нужно оставить 100. На каждом шаге модель обучается по
очереди без каждого из оставшихся признаков (при 1000 признаках — около
1000 обучений), и убирается тот, при исключении которого качество
модели улучшилось больше всего (или упало меньше всего). Процедура
повторяется на оставшихся признаках, пока не останется нужное
количество.

**Forward selection (жадное добавление).** Обратный вариант: на каждом
шаге в итоговый набор добавляется признак, дающий наибольший прирост
качества.

Есть и комбинированная стратегия: сначала жадное добавление, пока оно
улучшает качество, затем, когда добавление перестаёт помогать — жадное
удаление уже накопленных признаков.

## Встроенные методы (Embedded methods)

Отбор признаков происходит как побочный эффект обучения самой модели —
по скорости встроенные методы обычно быстрее wrapper-методов, но
медленнее чисто фильтрационных.

- **L1-регуляризация (Lasso)** — обнуляет часть весов, тем самым
  автоматически выполняя отбор признаков (подробнее про L1/L2/ElasticNet
  — см. тему `linear-regression-gd`). В scikit-learn — `SelectFromModel`,
  обёрнутый вокруг модели с L1-регуляризацией, автоматически убирает
  признаки с нулевым/малым весом.
- **L0-регуляризация** — встречается реже, в основном в глубинном
  обучении; штрафует напрямую за количество ненулевых весов.
- **Feature importance деревьев и композиций деревьев** (impurity-based
  feature importance) — чем сильнее признак уменьшает неопределённость
  (impurity) при разбиениях, тем он важнее. Оценка через случайный лес
  считается в среднем более честной, чем веса линейной регрессии, хотя и
  не идеальной — случайный шумовой признак иногда всё равно оказывается
  на высоких позициях.
- **Встроенные методы отбора в бустингах** — у CatBoost и XGBoost есть
  собственные методы (например, `select_features`), позволяющие явно
  задать желаемое число итоговых признаков.

Отбор признаков на основе одной модели (например, быстрой логистической
регрессии или случайного леса) — отдельный этап feature engineering, и
полученный набор признаков затем можно использовать с совершенно другой
моделью (не обязательно с той, что производила отбор).

## Permutation Importance

Идея: взять один столбец признака и случайным образом перемешать в нём
значения (тем самым «сломав» связь признака с объектами), затем
посмотреть, как изменилось качество модели без переобучения:

- если качество не изменилось (или почти не изменилось) — признак
  практически бесполезен, его можно удалять;
- если качество заметно упало — признак важен;
- если качество вдруг **улучшилось** после перемешивания «случайного
  мусора» в столбце — это тревожный знак: скорее всего, с моделью или
  данными что-то не так (возможна утечка данных), и стоит разобраться, а
  не радоваться такому улучшению.

Метод прост, но при этом весьма эффективен.

## Алгоритм Boruta

Модификация идеи permutation importance, устроенная как своеобразное
«состязание»:

1. Для каждого исходного (реального) признака создаётся его «теневая»
   копия — тот же столбец, но со случайно перемешанными значениями.
2. Реальные и теневые признаки объединяются в один датасет, на нём
   обучается модель, оценивается важность (feature importance) всех
   признаков.
3. В итоговый набор оставляют только те реальные признаки, важность
   которых оказалась выше важности их теневого «двойника».
4. Старые теневые признаки удаляются, генерируются новые, процесс
   повторяется несколько раз.

Реализован в библиотеке `boruta` для Python. В демонстрационном примере
из полного набора признаков алгоритм оставил всего 4, при этом качество
модели составило 0.87.

## Методология проведения экспериментов с отбором признаков

Ключевой принцип: нельзя применять все методы отбора «одним махом» в
надежде, что модель сама разберётся. Правильный подход — брать один
метод за раз, применять его, фиксировать изменение качества модели,
анализировать причины — и только потом переходить к следующему методу.
Эксперименты должны быть максимально независимы друг от друга.

Поскольку часть методов (в первую очередь wrapper) весьма затратна по
времени, не обязательно проводить эксперименты на полном датасете: можно
взять представительную подвыборку (например, 50-100 тысяч записей вместо
миллиона), предварительно проверив статистическим тестом её
репрезентативность, — это ускоряет процесс в разы без потери качества
выводов.

**Отбор признаков для ускорения инференса.** Помимо повышения качества,
отбор признаков может применяться исключительно ради ускорения
инференса: если качество тяжёлой модели, например, 0.67 по метрике, а
бизнес устраивает качество на уровне 0.6, можно убрать часть признаков
и получить модель, работающую в разы быстрее, пожертвовав частью
качества. Такой отбор делают уже на финальном этапе, когда модель для
продакшна практически определена.

**Рекомендуемый общий пайплайн** (из лекции): сначала — отбор по
корреляции (убрать явно слабо коррелирующие признаки), затем — обучить
простую модель со встроенными механизмами отбора (L1/L2-регуляризация
или feature importance для деревьев/композиций) и убрать наименее
значимые признаки. И только если этого недостаточно, а время и ресурсы
позволяют — переходить к оберточным методам как к самому долгому и
дорогому этапу; на практике до него часто просто не доходят.

## Код: практический пайплайн отбора признаков

Пример из семинара `lesson_03` — предсказание уровня насильственных
преступлений по социально-демографическим признакам (датасет Communities
and Crime): масштабирование → отбор по дисперсии (`VarianceThreshold`) →
отбор по корреляции с таргетом (`SelectKBest`) → встроенный отбор через
L1-регуляризацию (`SelectFromModel` + `Lasso`), всё внутри одного
`Pipeline` с перебором гиперпараметров через `GridSearchCV`:

```python
from sklearn.linear_model import Lasso, LinearRegression
from sklearn.metrics import mean_squared_error
from sklearn.model_selection import GridSearchCV, train_test_split
from sklearn.preprocessing import MinMaxScaler, StandardScaler
from sklearn.feature_selection import VarianceThreshold
from sklearn.feature_selection import SelectKBest, f_regression
from sklearn.feature_selection import SelectFromModel
from sklearn.pipeline import Pipeline
import pandas as pd
```

```python
# отбор признаков на основе дисперсии
vs_transformer = VarianceThreshold(0.01)

X_train_var = pd.DataFrame(
    data=vs_transformer.fit_transform(X_train_scaled),
    columns=X_train_scaled.columns[vs_transformer.get_support()],
)
X_test_var = pd.DataFrame(
    data=vs_transformer.transform(X_test_scaled),
    columns=X_test_scaled.columns[vs_transformer.get_support()],
)
```

```python
# отбор признаков на основе корреляции с целевой переменной
# Выбираем 15 лучших признаков
sb = SelectKBest(f_regression, k=15)

X_train_kbest = pd.DataFrame(
    data=sb.fit_transform(X_train_var, y_train),
    columns=X_train_var.columns[sb.get_support()],
)
X_test_kbest = pd.DataFrame(
    data=sb.transform(X_test_var), columns=X_test_var.columns[sb.get_support()]
)
```

```python
# отбор на основе регрессии с L1-регуляризацией
lasso = Lasso(5.0)
l1_select = SelectFromModel(lasso)

X_train_l1 = pd.DataFrame(
    data=l1_select.fit_transform(X_train_var, y_train),
    columns=X_train_var.columns[l1_select.get_support()],
)
X_test_l1 = pd.DataFrame(
    data=l1_select.transform(X_test_var),
    columns=X_test_var.columns[l1_select.get_support()],
)
```

```python
# весь пайплайн целиком, с перебором гиперпараметров
pipe = Pipeline(
    steps=[
        ("scaler", MinMaxScaler()),
        ("variance", VarianceThreshold(0.01)),
        ("selection", SelectFromModel(Lasso(5.0))),
        ("regressor", LinearRegression()),
    ]
)

pipe.fit(X_train, y_train)

param_grid = {
    "variance__threshold": [0.005, 0.0075, 0.009, 0.01, 0.011, 0.012],
    "selection__estimator__alpha": [0.5, 1.0, 1.5, 2.0, 5.0, 10.0]
}
grid_search = GridSearchCV(pipe, param_grid, cv=5, verbose=True)
grid_search.fit(X_train, y_train)

pipe_best = grid_search.best_estimator_
```

## Метод главных компонент (PCA)

В отличие от отбора признаков, PCA не выбрасывает исходные признаки, а
строит **новые** признаки на основе старых (автоматизированный feature
engineering) — новых признаков сильно меньше, чем исходных, но они
по-прежнему описывают те же объекты. Новые признаки — линейная комбинация
старых.

**Два упрощения задачи, которые делает классический PCA**: (1)
игнорируется целевая переменная — метод не смотрит на ответ вообще; (2)
строится именно линейная комбинация признаков. Игнорирование таргета на
первый взгляд кажется странным, но на практике не так плохо: данные
часто имеют внутреннюю структуру меньшей размерности, никак не связанную
с целевой переменной, поэтому оптимальные новые признаки можно строить,
не глядя на ответ.

### Постановка задачи

Есть исходные числовые признаки размерности m, хотим получить новые
числовые признаки размерности d, где d значительно меньше m. Требования:
(1) новые признаки линейно выражаются через исходные (переход от старого
базиса к новому); (2) при переходе должно быть потеряно наименьшее
количество исходной информации.

**Геометрическая интерпретация.** PCA — проекция m-мерного пространства
признаков на d-мерное подпространство. Если точки в двумерном
пространстве проецируются на прямую (переход из 2D в 1D), при удачно
выбранной прямой проекция сохранит значительную часть исходного
разброса данных, а при неудачно выбранной — большинство точек
«схлопнется», и информация о разбросе будет потеряна.

**Формализация требования.** Хотим, чтобы дисперсия проекции выборки на
новый признак была максимальной — чем больше дисперсия в новых
признаках, тем больше исходной информации сохранено. Идея PCA — подобрать
такую проекцию, чтобы дисперсия была максимально возможной среди всех
возможных проекций. Другая, эквивалентная интерпретация: поиск главных
компонент — это поиск подпространства меньшей размерности, где квадрат
отклонений проекций точек до этого подпространства минимален.

### Механизм (по шагам)

1. **Стандартизация данных (центрирование).** Перед применением PCA
   данные центрируются — из каждого признака вычитается его среднее
   значение (для метода из репозитория — ещё и делится на стандартное
   отклонение, приводя признак к среднему 0 и стандартному отклонению 1).
   Центрирование важно потому, что PCA чувствителен к масштабу признаков.
2. **Вычисление матрицы ковариаций** — PCA вычисляет ковариацию между
   признаками, чтобы понять, как они связаны друг с другом (растут ли
   вместе, обратно ли связаны и т. д.).
3. **Нахождение главных компонент.** Метод находит направления, вдоль
   которых данные «распылены» больше всего (максимальная дисперсия) —
   вычисляются собственные векторы и собственные значения ковариационной
   матрицы. Первая компонента (PC1) — направление с наибольшей
   дисперсией; вторая (PC2) — перпендикулярная первой, с наибольшей
   оставшейся дисперсией, и так далее. Компоненты должны быть
   ортогональны друг другу и иметь единичную длину (нормированы).
4. **Выбор топ-компонент и проекция данных.** После ранжирования
   компонент по значимости выбираются первые несколько (например, так,
   чтобы они покрывали ~95% дисперсии), и исходные данные проецируются на
   пространство, образованное этими компонентами.

### Математический вывод (Лагранжиан)

Пусть есть N отцентрированных точек (их среднее равно нулю), и вектор v
(единичной длины), на который они проецируются. Длина проекции i-й точки
— скалярное произведение точки на v. Дисперсия вдоль направления v
раскладывается через матрицу ковариации C и записывается как
произведение (v в квадрате, умноженное на C) — то есть как квадратичная
форма v-транспонированное умножить на C умножить на v.

Задача — максимизировать эту квадратичную форму при ограничении на
единичную длину вектора v (v-транспонированное умножить на v равно 1).
Составляется функция Лагранжа: квадратичная форма минус λ, умноженное на
(норма v в квадрате минус 1), где λ — множитель Лагранжа. Приравнивая
производные по v к нулю, получаем, что искомый вектор v должен
удовлетворять уравнению Cv = λv — то есть **v является собственным
вектором матрицы ковариации C**, а соответствующее λ — её собственным
значением. Первая главная компонента — собственный вектор с наибольшим
собственным значением, вторая — со следующим по величине, и так далее.

### Сколько компонент оставлять

Единой формулы нет — эмпирическое правило (например, если признаков
больше, чем объектов в выборке, можно ограничиться числом компонент
порядка половины числа объектов). Практический подход: собственные
значения матрицы ковариации упорядочены по убыванию. Можно посчитать
долю объяснённой дисперсии для k первых компонент (сумма первых k
собственных значений, делённая на сумму всех) и долю необъяснённой
дисперсии (единица минус эта доля). Если построить график зависимости
доли необъяснённой дисперсии от числа компонент, он обычно сначала резко
падает, а затем выходит на плато — точка «излома» (критерий крутого
склона, scree criterion) обычно и берётся как подходящее число
компонент.

### Интерпретируемость и ограничения

После применения PCA компоненты обычно очень трудно интерпретировать —
непонятно, что именно означает та или иная новая компонента. Известное
исключение — PCA, применённый к изображениям лиц (**eigenfaces**): первые
компоненты (в порядке убывания значимости) соответствуют самым значимым
чертам лица (глаза, нос, рот, овал), а каждая следующая добавляет всё
более мелкие детали. Преобразование можно обратить и восстановить
исходное изображение по ограниченному числу компонент (частично
сжать/разжать данные), хотя на практике для сжатия изображений
применяются другие алгоритмы.

Ограничения PCA:
- новые компоненты — линейные комбинации исходных признаков, поэтому
  интерпретировать их бывает сложно;
- PCA чувствителен к масштабу признаков — нужно обязательно
  стандартизовать данные заранее;
- при слишком сильном уменьшении размерности могут потеряться важные
  детали (возможна потеря информации);
- предполагается, что важная информация — в линейных комбинациях
  признаков; если связи между признаками нелинейные, PCA может быть
  неэффективен (тогда нужен Kernel PCA, t-SNE и т. п.).
- имеет смысл использовать, когда в данных есть чётко выраженная
  корреляция между признаками — если признаки уже слабо коррелируют
  между собой, PCA не даст значимого сжатия без потери дисперсии.

## Сингулярное разложение (SVD) и связь с PCA

PCA приходит ровно к тому же результату, что и **сингулярное разложение**
(singular value decomposition, SVD) — разложение матрицы данных X в
произведение трёх матриц: U · Σ · V-транспонированное, где U и V —
ортогональные матрицы (отвечают за поворот пространства), а Σ —
диагональная матрица с сингулярными числами на диагонали (сингулярные
числа — корни из собственных значений матрицы X-транспонированное на X).

Связь с PCA: столбцы матрицы V — собственные векторы матрицы
X-транспонированное на X, то есть это и есть главные компоненты. Столбцы
матрицы U·Σ — новые признаки (проекции исходных данных на главные
компоненты). Для снижения размерности до k измерений берут первые k
столбцов U и верхний k×k блок Σ — получившаяся матрица U_k · Σ_k содержит
k новых признаков, соответствующих первым k главным компонентам.

**PCA или SVD?** Оба метода приводят к одному результату, вопрос — в
вычислительном удобстве. У PCA (прямое нахождение собственных значений
матрицы XᵀX) есть вычислительные трудности; для SVD существуют
эффективные итерационные алгоритмы прямого вычисления разложения (без
явного нахождения собственных значений). На практике для больших
наборов данных SVD обычно работает быстрее, но однозначного правила нет.
SVD также активно применяется в рекомендательных системах (отдельная
тема курса).

## Многомерное шкалирование (MDS) и t-SNE — для визуализации

PCA (и SVD) можно использовать не только для сжатия признакового
пространства перед обучением модели, но и для **визуализации**:
многомерные данные сводятся к двум-трём измерениям, что даёт возможность
графического представления, помогает упрощать анализ, визуально выявлять
кластеры и обнаруживать выбросы. Но у PCA как средства визуализации есть
предел — метод линеен, и если структура данных существенно нелинейна,
двумерная PCA-проекция может плохо отражать реальную структуру. Для этого
есть специализированные нелинейные методы, которые обычно используют
только для визуализации, а не как признаки для дальнейшего обучения
модели.

### Multidimensional Scaling (MDS)

Идея — минимизация квадратов отклонений между исходными и новыми
попарными расстояниями между объектами. Целевая функция MDS (raw stress)
— сумма по всем парам объектов от квадрата разности между исходным
попарным расстоянием и расстоянием между теми же объектами уже во
встроенном (пониженном) пространстве.

### t-SNE (t-distributed stochastic neighbor embedding)

Нелинейный метод, ориентированный на **сохранение локальной структуры**
(в отличие от MDS, которое старается сохранить именно расстояния): важно
не сохранение абсолютных расстояний между объектами, а сохранение
пропорций/соседства между ними, за счёт чего t-SNE имеет меньшую
склонность сжимать точки в центр, чем другие алгоритмы. Особенно удобен,
когда нужно «увидеть» кластеры в данных.

Метод минимизирует дивергенцию Кульбака-Лейблера (KL-divergence) между
совместными вероятностями похожести объектов в исходном и во встроенном
(пониженном) пространствах — с помощью градиентного спуска.

Недостатки t-SNE:
- вычислительно дорог — на датасетах размером в миллионы объектов
  вычисление может занимать часы, тогда как PCA завершится за секунды
  или минуты;
- алгоритм стохастический: при разных начальных условиях может приходить
  к разным локальным минимумам KL-дивергенции — полезно запускать
  алгоритм с разными начальными значениями и выбирать вариант с
  наименьшей дивергенцией;
- глобальная структура данных явно не сохраняется (расстояния между
  далёкими кластерами на t-SNE-визуализации неинформативны) — этот
  недостаток частично устраняется инициализацией точек методом PCA
  (`init='pca'`).

## Код: PCA на практике, kernel PCA и t-SNE

Демонстрация PCA на синтетических двумерных данных — метод корректно
находит направление наибольшей дисперсии и не зависит от поворота данных
(поскольку работает с ковариационной матрицей, а не с исходными осями):

```python
import numpy as np
import matplotlib.pyplot as plt
from sklearn.decomposition import PCA

def PCA_show(dataset):
    plt.scatter(*zip(*dataset), alpha=0.5)

    dec = PCA()
    dec.fit(dataset) # обучаем PCA
    ax = plt.gca()
    for comp_ind in range(dec.components_.shape[0]): # пробегаемся по выделенным компонентам (осям)
        component = dec.components_[comp_ind, :]     # компонента
        var = dec.explained_variance_[comp_ind]      # объясненная дисперсия
        start, end = dec.mean_, component * var
        ax.arrow(start[0], start[1], end[0], end[1],
                 head_width=0.2, head_length=0.4, fc='r', ec='r')

    ax.set_aspect('equal', adjustable='box')
```

На нелинейных данных (концентрические круги, полумесяцы, кластеры-blobs)
PCA находит направления, которые не являются хорошими признаками — метод
хорошо работает только тогда, когда «главное направление изменчивости»
действительно линейно описывает данные; для сильно нелинейных многообразий
нужны kernel PCA, t-SNE, UMAP и подобные методы.

**PCA на лицах (eigenfaces) — сжатие и восстановление изображений:**

```python
from sklearn.datasets import fetch_olivetti_faces

faces = fetch_olivetti_faces(shuffle=True, random_state=432540)
faces_images = faces.data
faces_ids = faces.target
image_shape = (64, 64)

mean_face = faces_images.mean(axis=0)

red = PCA()
faces_images -= mean_face
red.fit(faces_images)
```

```python
# сжатие в N раз и восстановление
base_size = min(image_shape[0] * image_shape[1], faces_images.shape[0])

def compress_and_show(compress_ratio):
    red = PCA(n_components=int(base_size * compress_ratio))
    red.fit(faces_images)

    faces_compressed = red.transform(faces_images) # проецируем точки на выделенные оси
    faces_restored = red.inverse_transform(faces_compressed) + mean_face # обратное преобразование

    plt.figure(figsize=(16, 8))
    rows, cols = 2, 4
    n_samples = rows * cols
    for i in range(n_samples):
        plt.subplot(rows, cols, i + 1)
        plt.imshow(faces_restored[i, :].reshape(image_shape), interpolation='none',
                   cmap='gray')
        plt.xticks(())
        plt.yticks(())

compress_and_show(0.5)
```

Использование PCA-признаков вместо исходных пикселей может давать более
высокое качество классификации (проверялось через `GridSearchCV` на
`RandomForestClassifier`, обучая один раз на сырых пикселях, второй раз —
на первых 100 главных компонентах).

**Kernel PCA** — поскольку PCA фактически работает не с исходными
признаками, а с матрицей их ковариаций (по сути — со скалярными
произведениями), можно заменить обычное скалярное произведение
произвольным ядром K(xᵢ, xⱼ) — это соответствует переходу в другое
пространство, где предположение о линейности уже может иметь смысл
(аналогично kernel trick в SVM, см. тему `svm`). Проблема — заранее
непонятно, как подбирать ядро:

```python
from sklearn.decomposition import KernelPCA

def KPCA_show(X, y):

        dict(kernel='cosine', gamma=10),
    ]

    for i, p in enumerate(kernels_params):
        dec = KernelPCA(**p)
```
