## От n-грамм к нейросетевой генерации текста

N-граммная модель генерации текста — обычная вероятностная модель, где
следующий токен зависит только от ограниченного подмножества предыдущих
токенов. Это ограничивает качество генерации: чем больше предыдущего
текста модель способна учитывать, тем лучше генерация. Идея нейросетевых
моделей — учитывать все предыдущие токены, а не ограниченное окно.

Общий вид архитектуры для генерации текста: на вход подаётся текст, он
обрабатывается и превращается в вектор признаков; к вектору применяется
линейный слой, преобразующий размерность в размер словаря; на выход этого
слоя накладывается softmax, превращающий значения в вектор вероятностей
(сумма координат равна единице, каждая координата — вероятность
конкретного токена словаря). За один проход генерируется один токен, сеть
прогоняется многократно.

Свёрточные сети для этой задачи не подошли по двум причинам: во-первых,
свёртка работает медленно (нужно обработать всю входную последовательность
целиком, а рост числа свёрточных слоёв резко увеличивает время вычислений);
во-вторых, все входные токены участвуют в предсказании с одинаковым весом,
хотя реально токены ближе к текущей позиции обычно влияют на предсказание
сильнее, чем токены в начале последовательности.

## Рекуррентные нейронные сети (RNN)

RNN разработаны специально для последовательных данных: они обрабатывают
все токены входной последовательности один за другим. Сегодня RNN на
практике почти не используются для текста, но применяются во временных
рядах — архитектура, придуманная для текста, неожиданно оказалась полезной
и там.

Общая схема: один и тот же блок RNN (с одними и теми же весами) применяется
многократно. Первый блок принимает начало последовательности и предсказывает
следующее слово; каждый следующий блок принимает предыдущее скрытое
состояние и текущее слово и возвращает обновлённое состояние (делает
предсказание для текущего шага). Так можно обработать текст произвольной
длины, повторно применяя один и тот же блок.

### Устройство блока RNN

Блок принимает вектор предыдущего скрытого состояния h и векторное
представление (эмбеддинг) текущего токена x. Вектор h пропускается через
линейный слой с матрицей весов U, вектор x — через линейный слой с
матрицей весов W; результаты складываются, и сумма проходит через функцию
активации (сигмоида или гиперболический тангенс) — так получается
обновлённый вектор скрытого состояния h_t. Если нужен ещё и выход сети
(предсказание), поверх скрытого состояния накладывается дополнительный
линейный слой с матрицей весов V и смещением, преобразующий размерность
вектора в размер словаря — получается вектор логитов.

### Обучение RNN

Обучение — максимизация произведения вероятностей каждого следующего
токена при условии всех предыдущих (нужно поднять вероятность правильного
токена и понизить вероятность остальных). Чтобы превратить задачу
максимизации в минимизацию, к произведению вероятностей применяется
логарифм со знаком минус — получается функция ошибки, **кросс-энтропия**;
для всего корпуса кросс-энтропия усредняется по всем текстам. Итоговая
ошибка по всей последовательности — сумма ошибок по каждому блоку.

Главная проблема при расчёте градиентов — градиент по матрице U:
поскольку h_t зависит от предыдущего скрытого состояния h_(t-1), которое
само зависит от той же матрицы U, по правилу дифференцирования сложной
функции (chain rule) производная раскладывается рекурсивно по всем
предыдущим шагам — получается сумма произведений производных вида
dh_t/dh_(t-1), умноженных на производную по U на каждом шаге.

### Взрыв и затухание градиента

В этой сумме произведений скрыты две проблемы.

**Взрыв градиента.** Если норма произведения производных больше единицы, то
чем больше шагов, тем больше накапливается произведений — градиент
становится огромным, происходит переполнение, и в PyTorch появляются
значения NaN; модель перестаёт обучаться. Решения: регуляризация модели,
уменьшение learning rate, и, главное, **gradient clipping** (обрезка
градиента):
- «правильный» способ — нормирование: если норма градиента превышает
  заданный максимум (max_norm), градиент умножается на минимум из единицы
  и отношения max_norm к текущей норме — так контролируется длина вектора
  градиента, но не его направление;
- «ленивый» способ — покомпонентная обрезка всех координат градиента до
  заданного интервала (например, от −c до c).

**Затухание градиента.** Если функция активации — сигмоида (как чаще всего
и бывает), градиенты чаще склонны затухать, а не взрываться. Размер
затухания зависит от количества произведений в цепочке, то есть от
расстояния между токенами в тексте: для далёких друг от друга токенов
градиент получается маленьким, для близких — более значимым. Из-за этого
RNN хорошо учитывает зависимости между близкими словами и плохо — между
далёкими. Это следствие самой природы рекуррентной сети, которое нельзя
исправить простыми инженерными трюками — нужно менять архитектуру. Для
этого и придуманы LSTM и GRU.

## LSTM (Long Short-Term Memory)

Чтобы модель не «забывала» далёкие токены, в базовый блок RNN добавили
вектор памяти (**cell state**) — специальный вектор, переносимый от шага к
шагу, а вместе с ним — три «фильтра» (гейта):
- **фильтр забывания (forget gate)** — линейный слой от предыдущего
  скрытого состояния и текущего входа, с сигмоидой на выходе (значения от
  0 до 1); поэлементно умножается на вектор памяти — координаты, близкие к
  0, «стирают» соответствующую информацию из памяти, координаты, близкие к
  1, — сохраняют её;
- **фильтр входа (input gate)** — считается по похожей формуле, но с
  гиперболическим тангенсом (значения от −1 до 1); извлекает из текущего
  входа информацию, которую нужно добавить в вектор памяти;
- **фильтр выхода (output gate)** — извлекает из вектора памяти именно ту
  информацию, которая нужна для предсказания на текущем шаге.

Пример на задаче классификации тональности текста: встретив маркерное слово
(«вкусное»), модель может «забыть» часть накопленной ранее информации
(фильтр забывания) и одновременно зафиксировать, что текст, скорее всего,
положительный (фильтр входа); если слово нейтральное, добавлять в память
нечего. Значения фильтров не заданы эвристикой заранее, а настраиваются
автоматически в процессе обучения — минимизацией функции ошибки.

Проблема LSTM: за счёт вектора памяти и трёх фильтров число параметров
внутри блока увеличивается примерно в четыре раза по сравнению с обычным
RNN — сеть работает медленнее и дороже и сильнее склонна к переобучению.

## GRU (Gated Recurrent Unit)

Модификация, сокращающая число параметров при сохранении способности
учитывать длинные зависимости: фильтр забывания и фильтр входа объединены
в один, то есть используются два фильтра вместо трёх (или четырёх
дополнительных слоёв, как в LSTM) — число весов внутри блока сокращается
почти вдвое.

GRU не является однозначно «лучше» LSTM: на большом корпусе текстов LSTM
обычно показывает чуть лучший результат (у неё больше весов, она способна
«выучить» больше данных); если текстов немного, лучше обычно показывает
себя GRU — за счёт меньшего числа параметров и меньшей склонности к
переобучению.

## Двунаправленные и многослойные RNN

**Двунаправленные RNN.** Смысл слова иногда меняется в зависимости от того,
какое слово идёт за ним, поэтому текст можно читать не только слева
направо, но и справа налево: одна независимая сеть читает текст в одну
сторону, другая — в обратную, их выходы конкатенируются и пропускаются
через линейный слой. Использует вдвое больше параметров. Для генерации
текста почти не помогает (в генерации нет смысла «читать назад»), а для
классификации часто даёт заметный прирост качества.

**Многослойные RNN.** Несколько рекуррентных сетей накладываются друг на
друга: выход одной сети подаётся на вход следующей вместе со своим скрытым
состоянием — позволяет извлекать всё более сложные признаки. Недостаток —
растёт число параметров, и глубокую сеть становится сложнее обучить (чаще
возникает затухание, а не взрыв градиента, поскольку градиент просто не
доходит до дальних слоёв). На практике больше двух слоёв в таких сетях
почти никто не использует.

## Механизм внимания и задача seq2seq

Механизм внимания и архитектура трансформера были предложены в статье
«Attention Is All You Need» (2017 год) — эта архитектура оказалась
настолько удачной, что дальнейшее развитие классических RNN во многом
остановилось.

Трансформер изначально разработан для задачи **seq2seq** (sequence-to-
sequence) — преобразования входной последовательности в новую (машинный
перевод, суммаризация, перенос стиля текста). Для решения такой задачи
применяется концепция из двух моделей: **энкодер** кодирует входную
последовательность, **декодер** принимает выход энкодера и генерирует
новую последовательность.

До появления трансформера эту задачу решали парой рекуррентных сетей:
RNN-энкодер кодировал вход блок за блоком, скрытое состояние последнего
блока передавалось декодеру как начальное состояние, декодер генерировал
продолжение. Главная проблема — **«бутылочное горлышко» (bottleneck)**
между энкодером и декодером: вся информация о входной последовательности
должна была сжиматься в один вектор фиксированной размерности, из-за чего
значительная часть информации о длинной последовательности терялась.

### Идея механизма внимания

Вместо того чтобы полагаться только на один сжатый вектор, декодеру
разрешают «смотреть» на произвольные выходы энкодера. На каждом шаге
декодер вычисляет **скоры** — меры близости (например, скалярное
произведение) между собственным текущим вектором и каждым скрытым вектором
энкодера. Скоры пропускаются через softmax, получается распределение
вероятностей по токенам входной последовательности; эти вероятности
используются как веса для взвешивания векторов энкодера — получается
**контекстный вектор**, который возвращается в декодер и учитывается при
генерации следующего токена вместе с основным скрытым состоянием.

Смысл: декодер получает возможность на каждом шаге выбирать, из каких
именно токенов входной последовательности извлекать информацию, вместо
того чтобы полагаться на один фиксированный вектор.

## Архитектура трансформера

Настоящий прорыв произошёл, когда механизм внимания довели до отдельной
самостоятельной архитектуры и вовсе отказались от рекуррентности.
Трансформер состоит из блоков **энкодера** и **декодера**; каждый крупный
«кубик» на схеме (Multi-Head Attention, Feed Forward) на самом деле — не
один слой, а композиция из линейных слоёв и функций активации.

### Блок энкодера

**1. Multi-Head Self-Attention.** Self-attention — модификация внимания,
при которой каждый токен последовательности может напрямую взаимодействовать
с любым другим токеном той же последовательности. Для каждого токена
создаются три вектора: **Query (запрос)** — то, что токен «запрашивает» у
остальных; **Key (ключ)** — информация о том, что содержится внутри
токена; **Value (значение)** — та информация, которая в итоге извлекается.

Формула self-attention (словами): берём softmax от произведения матрицы
Query на транспонированную матрицу Key, делённого на корень из размерности
вектора Key (нормировочная константа — технический приём, чтобы после
умножения дисперсия значений не «разлеталась»), и умножаем результат на
матрицу Value. Матрицы Q, K, V получаются применением отдельных линейных
слоёв (с разными весами) ко всей входной последовательности. Произведение
Q на транспонированную K даёт квадратную матрицу скоров значимости одного
токена для другого.

**Multi-Head Attention** — в отличие от обычного self-attention (который
просто смотрит на другие токены), позволяет смотреть на разные *аспекты*
одного и того же токена одновременно (например, для слова «светит» —
отдельно «что светит» и «как светит»). Вектора Q, K, V делятся на
несколько независимых частей («голов»), каждая извлекает свою информацию,
результаты по всем головам конкатенируются и пропускаются через ещё один
линейный слой.

**2. Positional Encoding (позиционное кодирование).** Self-attention сам по
себе не учитывает порядок токенов — каждый токен может смотреть на любой
другой, но «не знает», где какой стоит. К обычному эмбеддингу слова
добавляется эмбеддинг его позиции в тексте (обучается вместе с остальными
параметрами модели) — так получается итоговое представление токена.

**3. Feed Forward.** Полносвязная сеть из двух линейных слоёв (первый
увеличивает размерность, второй возвращает к исходной, между ними —
функция активации, обычно ReLU), обрабатывающая информацию после
self-attention независимо для каждого токена.

**4. Add & Norm.** Как в ResNet — skip-connection (значение до слоя
суммируется со значением после него, чтобы бороться с затуханием
градиента), плюс нормализация после сложения для стабилизации обучения.
Чисто технический блок.

Энкодер трансформера — это несколько раз повторённый блок из Multi-Head
Self-Attention, позиционного кодирования, Feed Forward и Add & Norm.

### Блок декодера

По сравнению с энкодером добавлены два новых элемента:

**Masked Multi-Head Self-Attention.** Поскольку декодер генерирует текст,
во время обучения ему нужно запретить «подсматривать» на токены после
текущего — иначе он бы обучился опираться на ещё не сгенерированное
будущее. К формуле внимания добавляется маскирующая матрица: для токенов
после текущего подставляется минус бесконечность, для токенов до
текущего — ноль, из-за чего веса будущих токенов после softmax обнуляются.

**Cross-Attention.** Позволяет декодеру получать информацию от энкодера:
векторы Query берутся из выходной последовательности (декодера), а Key и
Value — из входной последовательности (энкодера). По сути декодер
«запрашивает» у энкодера нужную для текущего шага информацию.

**Резюме по трансформеру:** задача seq2seq плохо решается парой RNN
из-за «бутылочного горлышка», а механизм внимания позволяет декодеру
«заглянуть» в любую часть выхода энкодера и обойти это узкое место. В
исходном виде (энкодер + декодер вместе, как в оригинальной статье)
архитектура в современной практике почти не применяется — гораздо чаще
применяются её «половинки» по отдельности.

## От трансформера к BERT и GPT

Задачи с текстом можно классифицировать по соотношению длины входа и
выхода:
- **many-to-many, одинаковая длина** (NER, part-of-speech tagging) —
  достаточно только **энкодера**;
- **many-to-many, разная длина** (суммаризация, перевод, перенос стиля) —
  нужен **декодер**;
- **many-to-one** (классификация, регрессия) — достаточно **энкодера**.

Опираясь на это наблюдение, две компании независимо предложили архитектуры
на основе «половинок» трансформера: OpenAI в 2018 году разработала **GPT**
— модифицированный декодер; Google примерно тогда же — **BERT** —
модифицированный энкодер.

### BERT (Bidirectional Encoder Representations from Transformers)

В основе — блок энкодера трансформера. Главное преимущество — модель можно
**предобучить** на большом массиве неразмеченных данных (модель учится
«понимать язык»), а затем быстро **дообучить** под конкретную локальную
задачу. Базовый BERT (2018) содержал 110 миллионов параметров и обучался на
16 гигабайтах текста (BooksCorpus + английская Википедия) 4 дня на 16
видеокартах.

**Предобучение BERT** состоит из двух задач, решаемых одновременно:
1. **Masked Language Modeling (MLM)** — случайно выбирается 15% токенов
   текста: 80% заменяются токеном [MASK], 10% — случайными токенами (чтобы
   модель не переобучалась на сам факт маскирования), 10% остаются без
   изменений. Модель должна восстановить исходные токены на этих позициях.
2. **Next Sentence Prediction (NSP)** — в модель подаются два склеенных
   текста (в 50% случаев второй реально продолжает первый, в 50% —
   случайный несвязанный текст); по представлению служебного токена [CLS]
   модель предсказывает, является ли второй текст логичным продолжением
   первого. Дополнительно используются **Segment Embeddings**, явно
   указывающие принадлежность токена к первому или второму тексту.

На бенчмарке для сравнения моделей классификации BERT на момент выхода
обошёл ближайшего конкурента на 7%. Позже выяснилось (пост-фактум, а не
заранее спроектировано), что разные слои BERT с ростом номера слоя
извлекают всё более сложные лингвистические признаки: ранние слои —
части речи, дальше — синтаксические составляющие (подлежащее, сказуемое),
затем — именованные сущности, затем — семантические роли и кореференция.

**Модификации BERT:** RoBERTa (Meta) — обучается дольше и на большем
объёме данных, с динамическим маскированием (маска для MLM генерируется
заново при каждой подаче примера); ALBERT — сокращает число параметров
через parameter sharing (шеринг весов между слоями) и факторизацию матрицы
эмбеддингов; также DistilBERT, DeBERTa, ELECTRA и другие. BERT-подобные
модели по-прежнему активно применяются для классификации, где они
значительно легче полноценных больших языковых моделей — предобработка
данных, детекция нежелательного контента и подобные задачи, где текст
достаточно однозначно делится на классы.

### GPT (Generative Pre-trained Transformer)

В основе — блок декодера трансформера, из которого убрали cross-attention
(поскольку энкодера, с которым нужно было бы «сверяться», больше нет).
Архитектура: эмбеддинги + позиционные эмбеддинги → несколько раз повторённый
блок из masked multi-head attention, feed forward и add & norm → линейный
слой и softmax, дающие распределение вероятностей по словарю.

Рост масштаба GPT-моделей: GPT-1 — около 117 миллионов параметров, обучение
на ~1.5 гигабайта данных; GPT-2 — около 1.5 миллиарда параметров, 40
гигабайт данных; GPT-3 — 175 миллиардов параметров, 45 терабайт текстовых
данных.

Ключевая особенность, благодаря которой «выстрелила» GPT: задача языкового
моделирования (генерации следующего токена) оказалась настолько
универсальной, что к ней можно свести практически любую другую задачу
(например, классификацию текста можно переформулировать как генерацию
индекса нужного класса).

## Дообучение больших моделей на практике

Большие модели почти никогда не обучают с нуля — берут готовую модель с
весами, обученную кем-то другим, и действуют по одной из стратегий:
- **Полное дообучение (fine-tuning)** всех весов — рискованно для моделей с
  миллиардами и триллионами параметров: локальный датасет обычно намного
  меньше, чем нужно, модель быстро переобучается, «запоминая» выборку.
- **Частичное дообучение** — заморозить большую часть весов (обычно ранние
  слои, отвечающие за общее понимание языка) и дообучать только «голову»
  модели или отдельные её части.
- **PEFT (Parameter-Efficient Fine-Tuning)** — методы, при которых
  обучается лишь небольшая часть весов, а основная часть остаётся
  замороженной.
- **RAG (Retrieval-Augmented Generation)** — вместо дообучения весов, в
  контекст готовой модели на лету подкладывается дополнительная релевантная
  информация — не обучение в строгом смысле, а способ «уточнить» ответ за
  счёт контекста.
- **Агентные системы** — модели, способные сами «собирать» нужную
  информацию и добавлять её в свой контекст без отдельной инфраструктуры
  поиска.

На практике для решения бизнес-задач чаще всего используется комбинация
этих подходов, а не один способ в чистом виде.

## Практика: основы PyTorch (код-эталон для упражнений)

Ноутбук семинара «Знакомство с PyTorch» не разбирает конкретно RNN/LSTM/
трансформеры, но даёт общий фундамент работы с фреймворком, который
используется при реализации любых нейросетей на курсе — тензоры,
автоматическое дифференцирование и цикл обучения.

### Тензоры

```python
import torch

# Создание тензоров
t = torch.zeros((5, 3))
t = torch.ones((2, 3, 4))
t = torch.randn((2, 3, 2, 4))  # из стандартного нормального распределения

# in-place операции (с подчёркиванием в конце имени метода)
t = torch.ones((2, 3))
t.zero_()  # обнуляет "на месте", не создавая новый тензор
```

### Автоматическое дифференцирование (autograd)

```python
x = torch.tensor(-2., requires_grad=True)
y = torch.tensor(5., requires_grad=True)
z = torch.tensor(-4., requires_grad=True)

q = x + y
f = q * z

f.backward()

print(f'df/dz = {z.grad}')
print(f'df/dy = {y.grad}')
print(f'df/dx = {x.grad}')
```

### Полносвязная сеть и цикл обучения

Полносвязный слой — комбинация линейного преобразования и функции
активации. Сеть задаётся как последовательность таких преобразований:

```python
import torch.nn as nn

NN = nn.Sequential(
    nn.Linear(1, 5, bias=True),
    nn.Tanh(),
    nn.Linear(5, 5, bias=True),
    nn.Tanh(),
    nn.Linear(5, 1, bias=True)
    )
```

Более гибкий способ — наследование от `nn.Module` с описанием прямого
прохода в методе `forward`:

```python
class Net(nn.Module):
    def __init__(self, dim=1):
        super(Net, self).__init__()

        self.fc1 = nn.Linear(dim, 5)
        self.tanh1 = nn.Tanh()

        self.fc2 = nn.Linear(5, 5)
        self.tanh2 = nn.Tanh()

        self.fc3 = nn.Linear(5, 1)
        self.tanh3 = nn.Tanh()

    def forward(self, x):
        x = self.fc1(x)
        x = self.tanh1(x)

        x = self.fc2(x)
        x = self.tanh2(x)

        x = self.fc3(x)
        x = self.tanh3(x)

        return x
```

Функция обучения — цикл по эпохам с пятью стандартными шагами (forward →
loss → zero_grad → backward → step):

```python
from tqdm.auto import tqdm

def train(model, X, y, criterion, optimizer, num_epoch):
    '''
    args:
        model - модель нейронной сети
        X и y - обучающая выборка
        criterion - функция потерь, взятая из модуля `torch.nn`
        optimizer - оптимизатор, взятый из модуля `torch.optim`
        num_epoch - количество эпох обучения. Т.е. число градиентных шагов,
                    которые будут выполнены для каждого объекта в выборке
    '''
    for t in tqdm(range(num_epoch)):
        # Вычислим предсказания нашей модели
        y_pred = model(X)

        # Посчитаем значение функции потерь на полученном предсказании
        loss = criterion(y_pred, y)

        # Обнулим посчитанные ранее значения градиентов
        optimizer.zero_grad()

        # Выполним подсчёт новых градиентов
        loss.backward()

        # Выполним шаг градиентного спуска
        optimizer.step()

    return model


criterion = torch.nn.MSELoss()
optimizer = torch.optim.Adam(NN.parameters(), lr=1e-2)

NN = train(NN, X, Y, criterion, optimizer, 30)
```
