## Кластеризация как задача обучения без учителя

**Обучение без учителя (unsupervised learning)** — класс методов машинного
обучения, в которых отсутствует целевая переменная и требуется восстановить
структуру в данных: данные на входе не размечены, а поскольку правильных
ответов нет, возникают проблемы с самой измеримостью качества (об этом
подробнее — в разделе про метрики).

**Кластеризация** — задача обучения без учителя, цель которой за счёт
внутренней информации об объектах выборки найти «похожие» объекты и
отнести их к одному классу. В англоязычной литературе для этого метода
можно встретить название unsupervised classification — классификация без
учителя.

Формально: даны объекты x1, ..., xl. Требуется выявить в данных K
кластеров — таких областей, что объекты внутри одного кластера будут
максимально похожи друг на друга, а объекты из разных кластеров —
максимально непохожи. То есть нужно построить алгоритм, который каждому
объекту x сопоставляет номер кластера (число от 1 до K).

**Кластеризация vs классификация.** В классификации на основе размеченной
обучающей выборки учатся восстанавливать зависимость между признаками и
уже известными классами. В кластеризации никакой разметки нет — классы
«открываются» алгоритмом через одно только признаковое описание объектов.

### Зачем нужна кластеризация: применение

- Если есть неразмеченные данные и не хочется размечать их самостоятельно,
  кластеризация позволяет выделить группы объектов автоматически.
- Поиск аномалий, выделение пользовательских сегментов, группировка
  документов/текстов, обработка геоданных.
- Утилитарный сценарий: например, в задаче предсказания стоимости
  недвижимости объекты условно относят к категориям вроде «эконом»,
  «элитка», «бизнес» — но такие категории достаточно условны (застройщики
  присваивают их по-разному). Лучше кластеризовать пространство объектов
  самостоятельно — получившиеся кластеры могут отличаться от изначально
  прописанных лейблов и оказаться содержательнее.
- Разбить пространство объектов на кластеры и для каждого кластера
  построить свою отдельную модель машинного обучения (если данных
  достаточно). При инференсе новый объект относят к ближайшему по
  расстоянию до центров кластеру и запускают для него соответствующую
  модель — своего рода детализация пространства объектов перед
  моделированием.
- Забавное нишевое применение: **сжатие изображений**. Каждый цвет пикселя
  — это набор интенсивностей RGB; k-means позволяет найти в данных меньше
  кластеров цветов, чем исходных, — похожие по расстоянию цвета сливаются
  в один кластер, тем самым сжимая изображение.

Важно помнить: алгоритм кластеризации не знает, чего вы хотите. Невозможно
гарантировать, что, например, при кластеризации текстов получится
разбиение именно по темам — нередко кластеры оказываются
неинтерпретируемыми, и нужно детально изучать каждый кластер, чтобы понять,
почему получились именно такие группы.

## Метод k-средних (K-means)

Один из самых популярных и часто первый пробуемый метод: если данные
достаточно простые и легко кластеризуются, метод неплохо срабатывает, и на
нём часто и останавливаются.

**Алгоритм:**
1. Задаётся число кластеров k, случайным образом выбираются k центров
   кластеров (центроидов).
2. Каждый объект относится к тому кластеру, к центру которого он ближе.
3. Пересчитывается новый центр каждого кластера — как среднее
   арифметическое координат всех объектов внутри кластера (центр масс).
4. Шаги 2-3 повторяются до сходимости — пока центроиды не перестанут
   значительно перемещаться (разница между старым и новым центроидом
   становится меньше порога).

**Целевая функция.** K-means направлен на выбор центроидов, которые
минимизируют суммарное квадратичное расстояние от каждого объекта до
центра его кластера (эту величину также называют инерцией). В качестве
расстояния обычно берут квадрат евклидова расстояния — не обязательно, но
удобно, потому что квадратичную функцию проще дифференцировать. Поскольку
функция квадратичная (выпуклая), у неё есть единственный минимум — метод
гарантированно сходится, движение центроидов рано или поздно
стабилизируется.

**Правило локтя (elbow method)** — способ выбора k: выбирают такое
значение k, при котором происходит значительное («на изломе графика»)
уменьшение внутрикластерного расстояния — дальнейшее увеличение k уже не
даёт заметного выигрыша.

**Ограничение.** K-means хорошо работает только тогда, когда в
распределении данных есть ярко выраженные, примерно выпуклые центры
кластеров, и все объекты сконцентрированы вокруг них. Для сложной,
невыпуклой геометрии (например, одно кольцо точек, окружающее другой
кластер) k-means не справляется — сколько центроидов ни добавляй, кольцо
не удаётся корректно отделить, потому что алгоритм по своей природе ищет
компактные, примерно выпуклые кластеры.

Недостаток k-means — случайная инициализация центров может приводить к
разным результатам кластеризации от запуска к запуску. Достоинство —
скорость: на каждой итерации пересчитываются только расстояния до центров
кластеров, что дёшево.

```python
from sklearn.cluster import KMeans

k_means = KMeans(n_clusters=3)
k_means = k_means.fit(X)
clusters = k_means.predict(X)

plt.scatter(X[:,0], X[:,1], c=clusters)
plt.scatter(k_means.cluster_centers_[:, 0], k_means.cluster_centers_[:, 1], color='red', marker='x', s=200)
plt.title('K-means clustering')
plt.show()
```

## Иерархическая кластеризация

Идея: представить данные в виде графа, где вершины — объекты, а рёбра —
расстояния между ними. Результат визуализируется в виде **дендрограммы**
— дерева, где корень — уникальный кластер, объединяющий все объекты, а
листья — кластеры из одного объекта.

Два вида построения иерархии:
- **Агломеративная** (снизу вверх) — на каждой итерации алгоритм объединяет
  два ближайших кластера в один; изначально каждый объект — отдельный
  кластер (для выборки размера N на первой итерации будет N кластеров).
  Шаг повторяется, пока всё не объединится в один общий кластер.
- **Дивизионная** (сверху вниз) — на каждой итерации один кластер
  разбивается на два более мелких.

**Расстояние между кластерами** можно считать разными способами (linkage):
- среднее расстояние между всеми парами объектов из двух кластеров
  (`average`);
- минимальное расстояние между объектами двух кластеров (`min` /
  `single`);
- максимальное расстояние между объектами двух кластеров (`max` /
  `complete`);
- **расстояние Уорда (Ward)** — между кластерами берётся прирост суммы
  квадратов расстояний объектов до центра кластера, получаемого в
  результате их объединения: объединяются те два кластера, которые дают
  наименьший такой прирост (то есть максимально «похожи» друг на друга по
  этому критерию).

Итоговое число кластеров выбирают по дендрограмме: увеличивая порог
расстояния (ось Y), можно наблюдать, как объекты постепенно склеиваются в
более крупные кластеры. Практический способ выбора порога — искать
значение, при котором небольшое смещение порога вверх/вниз не приводит к
сильной перестройке кластеров (устойчивая картина), либо искать резкий
скачок на дендрограмме. Если нет специализированных условий (например,
известно, что кластеров должно быть не более K), число кластеров можно
выбирать именно так; также полезно опираться на доменную область задачи.

```python
from scipy.cluster.hierarchy import dendrogram
from sklearn.cluster import AgglomerativeClustering

def plot_dendrogram(model, **kwargs):
    counts = np.zeros(model.children_.shape[0])
    n_samples = len(model.labels_)
    for i, merge in enumerate(model.children_):
        current_count = 0
        for child_idx in merge:
            if child_idx < n_samples:
                current_count += 1  # leaf node
            else:
                current_count += counts[child_idx - n_samples]
        counts[i] = current_count

    linkage_matrix = np.column_stack(
        [model.children_, model.distances_, counts]
    ).astype(float)

    dendrogram(linkage_matrix, **kwargs)


model = AgglomerativeClustering(distance_threshold=0, n_clusters=None)
model = model.fit(X)
plt.title("Hierarchical Clustering Dendrogram")
plot_dendrogram(model, truncate_mode="level", p=5)
plt.xlabel("Number of points in node (or index of point if no parenthesis).")
plt.show()
```

## DBSCAN (Density-Based Spatial Clustering)

Развивает идею кластеризации через выделение связных компонент на основе
плотности данных, а не расстояния до центроида, поэтому успешно справляется
со сложной (невыпуклой) геометрией, где k-means не работает. Плотность
объекта определяется как количество других точек выборки в шаре радиуса
эпсилон (ε) вокруг него.

**Гиперпараметры:**
- **eps (ε)** — радиус окрестности вокруг точки;
- **min_samples** — минимальное число точек, которое должно находиться в
  этой окрестности (включая саму точку), чтобы точка считалась «основной».

**Три типа точек:**
- **Основные (core points)** — в окрестности радиуса eps не меньше
  min_samples точек;
- **Граничные (border points)** — в окрестности меньше min_samples точек,
  но точка сама достижима из какой-то основной точки;
- **Шумовые / выбросы (noise points)** — не основные и не достижимы ни от
  какой основной точки.

**Алгоритм:**
1. Выбрать точку без метки.
2. Если в окрестности eps меньше min_samples точек — пометить как шумовую.
3. Иначе — точка основная: находим все точки, достижимые из неё (прямо или
   через цепочку других основных точек), объединяем их в один кластер.
4. Граничные точки относят к тому кластеру, куда попала ближайшая основная
   к ним точка.
5. Процедура повторяется, пока не будут пройдены все точки.

Чем меньше радиус eps, тем больше и мельче получаются кластеры; чем больше
eps — тем крупнее и малочисленнее кластеры.

**Плюсы:** сам определяет число кластеров (по модулю заданных eps и
min_samples); успешно справляется со сложными формами кластеров; «из
коробки» умеет находить шумовые точки (выбросы) — у k-means такой
возможности нет вовсе.

**Минусы:** более долгое время работы; чувствителен к настройке
гиперпараметров; плохо справляется с кластерами переменной плотности.

```python
from sklearn.cluster import DBSCAN

dbscan = DBSCAN(eps=0.2, min_samples=10)
clusters = dbscan.fit_predict(X)

plt.scatter(X[:,0], X[:,1], c=clusters)
plt.show()
```

**HDBSCAN** — иерархическая модификация DBSCAN, для случаев, когда обычный
DBSCAN не справляется с очень сложными формами данных, кластерами
переменной плотности или когда подбор гиперпараметров особенно затруднён.
Не требует явного eps — определяет плотность автоматически и строит
иерархию, выбирая устойчивые кластеры по стабильности; часть точек может
остаться без кластера, если они нестабильны (шум).

```python
import hdbscan

hdb = hdbscan.HDBSCAN(min_cluster_size=10)
hdb_labels = hdb.fit_predict(X)

plt.scatter(X[:, 0], X[:, 1], c=hdb_labels, cmap='Spectral', s=30)
plt.title("HDBSCAN (min_cluster_size=10)")
plt.show()

# Визуализация уверенности принадлежности к кластеру
plt.scatter(X[:, 0], X[:, 1], c=hdb.probabilities_, cmap='viridis', s=30)
plt.colorbar(label='Membership probability')
plt.title("HDBSCAN: уверенность кластеризации")
plt.show()
```

## Спектральная кластеризация (кратко)

Ещё один подход, основанный на графовом представлении данных (похоже на
идею иерархической кластеризации): объекты представляются в виде графа, где
рёбра отражают расстояния между ними, и кластеры выделяются на основе
расстояний в этом графе — объекты с небольшим расстоянием относятся к
одному кластеру, а объекты, расстояние до которых больше некоторого
порога, — к другому. На практике даёт неплохие результаты на сложных,
невыпуклых данных, наравне с DBSCAN.

## Графовый метод (минимальное остовное дерево)

Вся выборка представляется как полный граф, где в вершинах — объекты, а на
рёбрах — расстояние между ними.

**Алгоритм:**
1. Построить минимальное остовное дерево по алгоритму Прима или Краскала.
2. На основе гиперпараметра K (число кластеров) удалить K−1 самых «тяжёлых»
   (длинных) рёбер — в результате граф распадётся на K компонент связности,
   которые и станут кластерами.

## Метрики качества кластеризации

Поскольку в кластеризации нет правильных ответов в привычном смысле,
оценка качества устроена иначе, чем в классификации/регрессии — есть
собственное семейство метрик (аналогично отдельным семействам метрик для
ранжирования и для временных рядов).

Метрики делятся на две группы:
- **Внешние метрики** используют дополнительную информацию — истинные
  метки классов объектов (если они всё-таки известны), сравнивая с ними
  полученное разбиение на кластеры.
- **Внутренние метрики** оценивают качество кластеризации саму по себе,
  без какой-либо внешней информации о разметке.

### Внешние метрики

- **Rand Index (RI)** — доля пар объектов, для которых исходное разбиение
  на классы и полученное разбиение на кластеры согласованы (то есть либо
  оба раза в одном кластере/классе, либо оба раза в разных). Считается как
  отношение числа «согласованных» пар ко всем возможным парам объектов.
- **Adjusted Rand Index (ARI)** — нормированная версия RI, не зависящая от
  числа объектов N и числа кластеров: значение 1 означает, что разбиения
  совпадают, больше 0 — разбиения похожи, около 0 — разбиения случайны и
  не связаны друг с другом, меньше 0 — разбиения не похожи (на практике
  встречается редко).
- **Гомогенность (однородность)** — насколько каждый кластер состоит из
  объектов одного класса. Худший случай — распределение по кластерам
  никак не уменьшило неопределённость (энтропию) относительно исходных
  классов; лучший — каждый кластер содержит объекты только одного класса.
  Тривиально «лучшую» гомогенность можно получить, выделив каждый объект в
  отдельный кластер (поэтому гомогенность нельзя использовать в отрыве от
  остальных метрик).
- **Полнота (completeness)** — насколько объекты одного класса удалось
  собрать в один кластер целиком (обратная сторона гомогенности).
  Тривиально «лучшую» полноту можно получить, положив все объекты в один
  общий кластер.
- **V-мера** — среднее гармоническое гомогенности и полноты (аналог
  F-меры из классификации, где гомогенность и полнота — аналоги точности и
  полноты). Гомогенность, полнота и V-мера принимают значения от 0 до 1, но
  не нормализованы и зависят от числа кластеров: при большом числе
  кластеров и малом числе объектов лучше использовать ARI; при числе
  кластеров меньше 10 и объектов больше 1000 эта зависимость менее
  выражена, и метрики можно использовать без опасений.

### Внутренние метрики

- **Среднее внутрикластерное расстояние** — сумма расстояний между всеми
  парами объектов внутри одного и того же кластера, делённая на число
  таких пар. Чем кучнее (компактнее) кластеры, тем меньше эта величина —
  её минимизируют.
- **Среднее межкластерное расстояние** — аналогично, но считается для пар
  объектов из разных кластеров. Эту величину максимизируют — чем более
  отдалены кластеры друг от друга, тем лучше.
- **Индекс Данна (Dunn Index)** — отношение минимального межкластерного
  расстояния (по всем парам кластеров) к максимальному внутрикластерному
  расстоянию (по всем кластерам). Одновременно учитывает и компактность
  кластеров, и их разделённость; полезен, когда число кластеров уже
  известно.
- **Коэффициент силуэта (Silhouette)** — не требует знания истинных меток.
  Для каждого объекта считаются: a — среднее расстояние до всех объектов
  того же кластера; b — среднее расстояние до объектов ближайшего другого
  кластера. Силуэт объекта показывает, насколько относительная разница
  между b и a велика (нормированная так, чтобы значение лежало в
  интервале от −1 до 1). Силуэт выборки — средний силуэт по всем объектам:
  около 1 — чётко выраженные, хорошо разделённые кластеры; около 0 —
  кластеры перекрываются; около −1 — плохая, разрозненная кластеризация.
  Силуэт достигает наибольших значений на компактных, выпуклых по форме
  кластерах и может «привирать» для невыпуклых форм — поэтому при
  отсутствии разметки лучше ориентироваться на несколько метрик сразу.

**Как выбирать метрику на практике:**
- если известны истинные метки — лучше использовать V-меру или ARI;
- если известно число кластеров — хорошо подходит индекс Данна;
- если совсем ничего неизвестно — неплохо работает силуэт;
- если есть возможность собрать разметку в достаточном объёме — возможно,
  стоит вообще решать задачу классификации, а не кластеризации.

**Практическое применение метрик для подбора k.** Поскольку выбор числа
кластеров остаётся проблемой почти для всех алгоритмов, можно перебирать
разные значения k, для каждого считать значение метрики (например,
силуэта) и выбирать k, при котором метрика максимальна — системный подход
вместо интуитивного подбора «на глаз».

```python
from sklearn.metrics import silhouette_score, homogeneity_score, v_measure_score

silhouette = []
homogeneity = []
v_measure = []
for n_c in range(2, 8):
    k_means = KMeans(n_clusters=n_c)
    k_means = k_means.fit(X)
    clusters = k_means.predict(X)
    silhouette.append(silhouette_score(X, clusters))
    homogeneity.append(homogeneity_score(y, clusters))
    v_measure.append(v_measure_score(y, clusters))

plt.figure(figsize=(11, 8))
plt.plot(range(2,8), silhouette, label='silhouette')
plt.plot(range(2,8), homogeneity, label='homogeneity')
plt.plot(range(2,8), v_measure, label='v_measure')
plt.xlabel('n_clusters')
plt.ylabel('metric')
plt.legend(loc='best')
plt.show()
```

## Сравнение методов на разных данных

Разные алгоритмы кластеризации по-разному справляются с разной геометрией
данных (окружности, полумесяцы, кластеры разной плотности, кластеры
разного размера, отсутствие структуры вовсе). Общая рекомендация — не
полагаться на один метод, а сравнивать несколько алгоритмов (K-means,
Agglomerative Clustering, DBSCAN, HDBSCAN) на своих данных, используя
внутренние метрики (силуэт) там, где нет разметки:

```python
from sklearn import cluster, datasets
from sklearn.preprocessing import StandardScaler

n_samples = 1500
noisy_circles = datasets.make_circles(n_samples=n_samples, factor=.5, noise=.05)
noisy_moons = datasets.make_moons(n_samples=n_samples, noise=.05)
blobs = datasets.make_blobs(n_samples=n_samples, random_state=8)

X, y = noisy_moons
X = StandardScaler().fit_transform(X)

k_means = cluster.KMeans(n_clusters=2)
dbscan = cluster.DBSCAN(eps=.3)
average_linkage = cluster.AgglomerativeClustering(linkage="average", n_clusters=2)

for name, algorithm in (('KMeans', k_means), ('DBSCAN', dbscan), ('Agglomerative', average_linkage)):
    algorithm.fit(X)
    y_pred = algorithm.labels_.astype(int)
    plt.scatter(X[:, 0], X[:, 1], c=y_pred)
    plt.title(name)
    plt.show()
```

Итого рассмотренные алгоритмы кластеризации: k-means, иерархическая
(агломеративная) кластеризация, DBSCAN и HDBSCAN, спектральная
кластеризация, графовый метод на минимальном остовном дереве. Все
рассмотренные метрики (RI/ARI, гомогенность/полнота/V-мера, силуэт, индекс
Данна) применимы к любому из этих подходов и могут использоваться для
подбора гиперпараметров (например, через максимизацию силуэта).

## Визуализация многомерных данных (MDS, t-SNE)

Смежная задача — не сама кластеризация, а **визуализация** многомерного
пространства признаков, когда рисовать можно только в одном, двух или трёх
измерениях, но хочется, чтобы картинка максимально соответствовала
структуре исходного многомерного пространства.

**Многомерное шкалирование (MDS).** Считается расстояние между объектами в
исходном пространстве и расстояние между их проекциями в новом (например,
двумерном) пространстве. Координаты проекций подбираются так, чтобы
расстояния между проекциями были максимально похожи на исходные расстояния
— то есть сохранить пропорции исходных расстояний. Ограничение метода: если
пытаться сжать объекты из очень многомерного пространства в двумерное,
сохранить все попарные расстояния зачастую попросту невозможно.

**t-SNE.** Поскольку MDS не всегда может сохранить исходные расстояния при
проекции, t-SNE ослабляет требование: вместо сохранения самих расстояний
метод старается сохранить пропорции близости — далёкие объекты должны
остаться далеко, близкие — остаться близко. Алгоритм строит вероятностное
распределение близости точек в исходном пространстве (окрестность каждой
точки предполагается распределённой примерно нормально) и подбирает такое
расположение точек в целевом (2-3-мерном) пространстве, чтобы аналогичное
распределение близости в новом пространстве было максимально похоже на
исходное — похожесть двух распределений измеряется через дивергенцию
Кульбака-Лейблера (KL-дивергенцию). Для целевого пространства используется
распределение с более «тяжёлыми хвостами» (Коши), которое не так сильно
штрафует за увеличение расстояния между объектами, как нормальное.

На датасете MNIST (рукописные цифры) t-SNE заметно лучше визуализирует
похожие объекты, чем PCA: у PCA классы цифр перемешиваются и накладываются
друг на друга, тогда как t-SNE лучше разводит классы по сторонам (хотя
определённые наложения остаются и там). Если стоит задача визуализировать
многомерный объект, t-SNE обычно хороший выбор, хотя на сложных
многомерных структурах он может работать довольно медленно.

Кластеризация и методы снижения размерности/визуализации (PCA, MDS, t-SNE)
чаще всего выступают как предварительная обработка или вспомогательная
подзадача для более крупной задачи, но также применяются и как
самостоятельные инструменты.
