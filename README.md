# ml-coach

Персональный ML-тренажёр, собранный поверх материалов курса машинного
обучения (ВШЭ, автор — Евгений Паточенко и Жанна Смелкова) и публичного
репозитория
[github.com/evgpat/ml_basics_course](https://github.com/evgpat/ml_basics_course).

Идея простая: вместо того чтобы пересматривать лекции и решать задачи в
одиночку, вы открываете эту папку в [Claude Code](https://docs.claude.com/claude-code)
(или подключаете скилл в Claude Cowork — см. ниже) и говорите `/coach`,
`/case` или `/interview` — а дальше Claude сам подтягивает нужный материал
курса, задаёт вопросы и код-упражнения по вашему прогрессу, реально
выполняет ваш код для проверки и отслеживает, что нужно повторить.

Никакого сервера, аккаунта или облака — всё состояние (что вы уже прошли,
что нужно повторить, история кейсов и собеседований) хранится в обычных
файлах на вашем диске, в этой же папке.

## Как это работает

Это не отдельное приложение, а **Claude Code Skill** — набор markdown-файлов
с инструкциями (`.claude/skills/mlcoach/`), которые Claude Code
автоматически подхватывает, когда вы открываете эту папку. Когда вы
пишете `/coach`, `/case` или `/interview`, Claude Code даёт Claude
прочитать соответствующий файл-инструкцию и следовать ей: прочитать вашу
карту тем и прогресс, подтянуть нужный конспект, задать вопросы, дождаться
вашего ответа, реально выполнить ваш код (у Claude Code есть доступ к
терминалу), оценить решение и обновить файл прогресса.

Схематично:

```
Вы пишете /coach
        │
        ▼
Claude читает _coach/curriculum.yaml (карта тем) и _coach/progress.json (ваш прогресс)
        │
        ▼
Выбирает тему (новую по возрастанию сложности или просроченную для повторения)
        │
        ▼
Читает готовый конспект темы из content/notes/<тема>.md
        │
        ▼
Даёт recap своими словами + 1-3 задания (вопрос / код / найди ошибку)
        │
        ▼
Вы отвечаете → Claude реально выполняет ваш код и сверяет с эталоном
        │
        ▼
Обновляет _coach/progress.json: когда повторить эту тему в следующий раз
```

Ключевая особенность: материалы курса заранее собраны в готовые конспекты
(`content/notes/`) — по одному на тему, объединяющему лекцию, семинар и
материалы репозитория Евгения Паточенко в единый читаемый текст без формул
в "сыром" LaTeX. Это сделано один раз заранее, поэтому сама тренировочная
сессия не тратит время и токены на разбор PDF и ноутбуков заново каждый
раз — Claude просто читает готовый файл.

## Требования

Работает двумя способами:

- **[Claude Code](https://docs.claude.com/claude-code)** (CLI,
  IDE-расширения, десктоп-приложение, веб-версия `claude.ai/code`) —
  основной, полностью поддерживаемый путь. Скилл подхватывается
  автоматически из `.claude/skills/`, ничего дополнительно ставить не
  нужно.
- **Claude Cowork** (режим в десктоп-приложении Claude для работы с
  локальной папкой) — через загрузку готового архива скилла, см. раздел
  [Использование в Claude Cowork](#использование-в-claude-cowork) ниже.

В обоих случаях нужен доступ к выполнению кода на Python (`sklearn`,
`pandas` и т.д.) — и Claude Code, и Cowork умеют выполнять код в своей
среде.

**Не сработает:**
- в обычном чате/приложении **Claude** (claude.ai) без Cowork — это другой
  продукт, он не читает `.claude/skills/` и не поддерживает загрузку
  кастомных скиллов;
- в мобильном приложении Claude.

## Установка

1. Убедитесь, что у вас установлен [Claude Code](https://docs.claude.com/claude-code)
   (`npm install -g @anthropic-ai/claude-code` для CLI, либо скачайте
   десктоп-приложение/установите IDE-расширение — см. официальную
   документацию по ссылке).

2. Склонируйте этот репозиторий **вместе с submodule** (это важно — без
   флага ниже часть материалов не скачается):

   ```bash
   git clone --recurse-submodules https://github.com/Andryuha40/ml-coach.git
   cd ml-coach
   ```

   Если уже склонировали без флага — доскачайте submodule отдельно:

   ```bash
   git submodule update --init
   ```

   Submodule (`content/ml_basics_course/`, оригинальный репозиторий
   Евгения Паточенко) обязателен только для режима `case` — там нужны
   реальные датасеты, на которых выполняется код. Для `coach` и
   `interview` он не обязателен: весь нужный материал уже собран в
   `content/notes/`.

3. Откройте папку `ml-coach` в Claude Code (`claude` в терминале из этой
   папки, либо "Open Folder" в десктоп-приложении/IDE-расширении).

4. Напишите `/mlcoach`, `/coach`, `/case` или `/interview` — и начинайте.

Никакой дополнительной настройки, API-ключей отдельно от вашей обычной
подписки Claude Code или регистрации не требуется.

## Режимы

Все три режима реализованы как ветки одного скилла `mlcoach`
(`.claude/skills/mlcoach/`). Можно вызвать диспетчер (`/mlcoach`) и он
спросит, какой режим нужен, либо сразу назвать нужную команду.

### `coach` — точечная тренировка

```
/coach
```

Тренажёр сам решает, чем заниматься сегодня:
- если есть тема, которую вы недавно проходили и пора повторить (спустя
  1, 3 или 10-14 дней — в зависимости от того, насколько уверенно вы
  ответили в прошлый раз) — предложит именно её;
- иначе — возьмёт следующую по сложности новую тему (от простого к
  сложному: сначала основы, потом метрические методы и деревья, потом
  ансамбли и признаки, потом продвинутые темы вроде интерпретируемости и
  временных рядов, LLM/агенты/RAG — в конце).

Вы также можете назвать тему сами ("позанимайся со мной SVM").

Дальше Claude:
1. Коротко напоминает суть темы своими словами — причём не общий пересказ,
   а именно то, что понадобится в заданиях ниже.
2. Даёт 1-3 задания: концептуальный вопрос ("почему...", "что сломается,
   если..."), код-упражнение (реальный фрагмент кода курса с убранной
   реализацией — напишите сами) или "найди и объясни ошибку" (рабочий код
   с намеренной мутацией — багом).
3. Ждёт вашего ответа. Если это код — реально выполняет его и сравнивает
   поведение/метрику с эталоном, показывает traceback, если упало.
4. Даёт содержательный фидбек (не просто "верно/неверно") и обновляет
   прогресс.

### `case` — бизнес-кейс целиком

```
/case
```

Совсем другой формат: вместо наводящих вопросов вы получаете задачу так,
как её сформулировал бы реальный заказчик — без указания, что это за тип
задачи (регрессия? классификация?), какую метрику использовать и какую
модель брать. Всё это нужно решить самостоятельно: понять постановку,
выбрать подход, написать код, обосновать выбор метрики.

Дальше — ревью в духе код-ревью на работе: соответствует ли выбранная
метрика той асимметрии издержек ошибок, которая заложена в кейсе (например,
не пропустить ни одного больного важнее, чем не пугать здоровых лишний раз
— или наоборот), чиста ли методология (нет ли утечки данных, правильно ли
разбиты train/test, особенно для временных данных), заметили ли вы
этически чувствительные моменты, если они в кейсе есть (например,
использование признака, коррелирующего с полом или расой).

В репозитории сейчас 7 кейсов из разных доменов: недвижимость, кредитный
скоринг, контроль качества, маркетинг, ценообразование на сырьё,
распределение социальных программ, медицинский скрининг.

### `interview` — мок-собеседование

```
/interview
```

Имитация технического собеседования: вопрос за вопросом, без вступительных
объяснений (Claude сразу спрашивает, а не читает мини-лекцию), с
уточняющими follow-up вопросами, если ответ поверхностный — реальный
интервьюер не соглашается молча с первым более-менее правильным ответом,
он копает глубже. Может охватывать сразу несколько тем за одну сессию (вы
сами выбираете — одна тема, область, или "выбери сам"). В конце — честный
структурированный вердикт: что сильно, что слабо, примерная оценка уровня
по итогам именно этого собеседования, и какие темы стоит подтянуть через
`/coach`.

## Уровень подготовки (Junior/Middle/Senior/Pro)

Опционально можно задать уровень подготовки — тогда все три режима
подстроят сложность вопросов и заданий под него:

- **Junior** — базовые определения, больше наводящих подсказок, задания
  проще и с шаблоном структуры решения.
- **Middle** — стандартный уровень по умолчанию (если уровень не задан —
  всё работает как раньше, без изменений).
- **Senior** — вопросы на edge cases и trade-offs, задания объединяют
  несколько тем сразу, меньше подсказок.
- **Pro** — максимально открытые, недоопределённые задачи, ожидание, что
  вы сами поднимете неочевидные проблемы (этичность, устойчивость,
  продакшен-риски) без наводящих вопросов.

Задать или сменить уровень можно в любой момент, просто сказав об этом
("давай на уровне Senior", "я сейчас скорее Junior") — даже посреди уже
идущей сессии, не только в начале. Уровень сохраняется между сессиями (в
`_coach/progress.json`), пока вы явно не попросите его сменить.

## Структура репозитория

```
ml-coach/
├── .claude/skills/mlcoach/
│   ├── SKILL.md              # диспетчер: определяет, какая ветка нужна
│   ├── coach.md                # полная логика режима coach
│   ├── case.md                   # полная логика режима case
│   └── interview.md               # полная логика режима interview
├── content/
│   ├── lectures/               # PDF-слайды лекций 01-10 + текстовые транскрипты 11-19
│   ├── seminars/                # ноутбуки семинаров + текстовые транскрипты
│   ├── notes/                    # готовые консолидированные конспекты по каждой теме
│   └── ml_basics_course/          # git submodule → github.com/evgpat/ml_basics_course
└── _coach/
    ├── curriculum.yaml          # карта из 20 тем: источники, конспект, концепты, пререквизиты
    ├── cases.yaml                 # каталог из 7 бизнес-кейсов для case
    ├── progress.json               # ваш прогресс по темам + уровень подготовки
    ├── case_log.json                # история пройденных кейсов
    └── interview_log.json            # история собеседований
```

`_coach/curriculum.yaml` — это карта из 20 тем, от основ (введение в ML, линейная
регрессия) через метрические методы, деревья и ансамбли, отбор признаков и
интерпретируемость, до продвинутых тем (кластеризация, временные ряды,
трансформеры, LLM/агенты/RAG — этот последний блок специфичен именно для
этого курса, в репозитории Евгения Паточенко его нет). Плюс опциональный
капстоун-модуль с двумя сквозными бизнес-кейсами на реальных данных.

## О материалах и анонимизации

- PDF-слайды лекций 01-10 и текстовые транскрипты лекций 11-19/семинаров
  10-18 — авторские материалы курса. Транскрипты живых занятий были
  анонимизированы перед публикацией: упоминания конкретных студентов и их
  личные истории удалены или обезличены (переформулированы в общие вопросы
  "На подумать"), техническое содержание не менялось.
- `content/notes/` — по одному конспекту на тему, собранному из материалов
  репозитория Евгения Паточенко (основа) и уникальных локальных материалов
  курса (дополнение, например блок про LLM/агентов, которого в репозитории
  нет вообще) в единый файл — чтобы `coach` и `interview` не парсили
  PDF/ноутбуки заново на каждой сессии.
- `content/ml_basics_course/` — не копия, а git submodule на оригинальный
  репозиторий Евгения Паточенко (MIT license) — при обновлении апстрима
  просто обновите submodule (`git submodule update --remote`).

## Использование в Claude Cowork

В приложении Claude (десктоп) есть режим **Cowork** — работа с выбранной
локальной папкой — и отдельный раздел **Settings → Skills**, где можно
загрузить кастомный скилл (zip-архив с `SKILL.md` внутри). Это отдельный
от Claude Code продукт, но на практике оказался совместимым.

**1. Скачайте самодостаточный архив** — пересобирать или отдельно
клонировать репозиторий не нужно, в архиве уже есть всё для всех трёх
режимов: логика скилла, `curriculum.yaml`/`cases.yaml`, все 24
консолидированных конспекта (`content/notes/`) и 8 датасетов, на которых
строятся бизнес-кейсы `case` (≈2 МБ):

**[⬇ Скачать mlcoach-standalone.zip](https://raw.githubusercontent.com/Andryuha40/ml-coach/main/mlcoach-standalone.zip)**

**2. Распакуйте архив** в любую папку на диске — это будет ваша рабочая
папка, аналог склонированного репозитория (внутри та же структура:
`_coach/`, `content/notes/`, `content/ml_basics_course/datasets/`).

**3. Загрузите скилл.** В приложении Claude: **Settings → Skills → Add →
Upload a skill** → выберите файл `SKILL.md` из распакованной папки (или
сам архив — зависит от того, что примет загрузчик; если не сработает,
попробуйте перезаархивировать только распакованную папку целиком).

**4. Откройте Cowork и выберите рабочей папкой именно распакованную
папку.** Это важно: `_coach/curriculum.yaml` и остальные файлы читаются
по пути относительно рабочей папки.

**5. Напишите** `/coach`, `/case` или `/interview` в Cowork-сессии.

> Честно говоря, я не проверял на практике, что именно из шагов 3-4
> Cowork реально требует — возможно, для файлов скилла (`SKILL.md` и т.д.)
> достаточно загрузки в Settings, а данные (`_coach/`, `content/`) он
> видит только через выбранную рабочую папку; возможно, наоборот, всё
> содержимое архива становится доступно уже после загрузки в Skills, без
> выбора папки. Сделайте оба шага (3 и 4) — если что-то из них окажется
> лишним, ничего не сломается.

Cowork умеет выполнять код, поэтому проверка код-упражнений тоже должна
работать. Это не официально поддерживаемый путь (в отличие от Claude
Code), поэтому в случае проблем — переходите на Claude Code, там всё
гарантированно работает.

Если нужен только сам скилл без данных (например, вы уже склонировали
репозиторий целиком через `git clone` и просто хотите отдельно
установить скилл в Cowork) — есть более лёгкий
**[mlcoach.zip](https://raw.githubusercontent.com/Andryuha40/ml-coach/main/.claude/skills/mlcoach.zip)**
(14 КБ, только `SKILL.md`/`coach.md`/`case.md`/`interview.md`, без данных
— тогда рабочей папкой в Cowork должен быть склонированный репозиторий).

Если вы форкнули репозиторий и меняли `.claude/skills/mlcoach/`,
`_coach/curriculum.yaml` или конспекты — оба архива нужно пересобрать
самостоятельно и закоммитить заново. Для `mlcoach.zip`:

```bash
cd .claude/skills
zip -r mlcoach.zip mlcoach/          # macOS/Linux
# или в PowerShell: Compress-Archive -Path mlcoach -DestinationPath mlcoach.zip
```

Для `mlcoach-standalone.zip` — соберите папку с той же структурой, что и
`ml-coach/`, но только с нужными для `coach`/`case`/`interview` файлами
(`.claude/skills/mlcoach/*.md` → в корень; `_coach/curriculum.yaml`,
`_coach/cases.yaml` и пустые шаблоны `progress.json`/`case_log.json`/
`interview_log.json`; `content/notes/`; датасеты из `cases.yaml` —
`content/ml_basics_course/datasets/<то, что перечислено в cases.yaml>`),
и заархивируйте её тем же способом.

## Сброс прогресса / форк без своей истории

`_coach/progress.json`, `_coach/case_log.json` и `_coach/interview_log.json`
— это ваш личный прогресс. Если хотите поделиться своим форком репозитория
с кем-то ещё без своей истории — сбросьте их к пустому шаблону (внутри
каждого файла есть `_comment`, поясняющий структуру) перед публикацией.
