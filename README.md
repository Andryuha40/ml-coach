# ml-coach

Персональный ML-тренажёр для Claude Code: два режима практики поверх
материалов курса машинного обучения (ВШЭ, автор — Евгений Паточенко,
[github.com/evgpat/ml_basics_course](https://github.com/evgpat/ml_basics_course)).

- **`/coach`** — точечная прокачка конкретной темы: recap своими словами,
  1-3 задания (концептуальный вопрос / код-упражнение / найди ошибку),
  реальная проверка кода, прогресс со spaced-repetition (повтор через
  1/3/10-14 дней в зависимости от того, насколько уверенно был решён
  предыдущий заход).
- **`/case`** — сквозной бизнес-кейс на реальном датасете: задача выдаётся
  языком заказчика, без указания типа задачи/метрики/модели — это должен
  решить сам пользователь. Оценка — ревью в духе код-ревью на работе:
  соответствует ли выбранная метрика названной в кейсе асимметрии издержек
  ошибок, чиста ли методология, смог бы пользователь объяснить результат
  нетехническому человеку.

## Установка

```bash
git clone --recurse-submodules <URL этого репозитория>
cd ml-coach
```

Если склонировали без `--recurse-submodules`:

```bash
git submodule update --init
```

Откройте папку репозитория в Claude Code — скиллы `/coach` и `/case`
подхватятся автоматически из `.claude/skills/`.

## Структура

```
ml-coach/
├── .claude/skills/
│   ├── coach/SKILL.md        # логика режима /coach
│   └── case/SKILL.md         # логика режима /case
├── content/
│   ├── lectures/              # PDF-слайды (01-10) + текстовые транскрипты (11-19)
│   ├── seminars/               # ноутбуки семинаров + текстовые транскрипты (10-18)
│   └── ml_basics_course/       # git submodule → github.com/evgpat/ml_basics_course
└── _coach/
    ├── curriculum.yaml         # карта тем для /coach: источники, концепты, пререквизиты
    ├── cases.yaml               # каталог бизнес-кейсов для /case
    ├── progress.json            # ваш прогресс по темам (создаётся пустым)
    └── case_log.json            # история пройденных кейсов (создаётся пустой)
```

`progress.json` и `case_log.json` — это ваш личный прогресс. Если хотите
делиться репозиторием дальше без своей истории, сбросьте их к шаблону
(смотри `_comment` внутри файлов) перед публикацией форка.

## О материалах

- PDF-слайды лекций 01-10 и текстовые транскрипты лекций 11-19/семинаров
  10-18 — авторские материалы курса. Транскрипты живых занятий были
  анонимизированы перед публикацией: упоминания конкретных студентов и их
  личные истории удалены или обезличены (переформулированы в общие вопросы
  "На подумать"), техническое содержание не менялось.
- `content/ml_basics_course` — не копия, а git submodule на оригинальный
  репозиторий Евгения Паточенко (MIT license) — при обновлении апстрима
  просто обновите submodule.

## Использование

```
/coach
```
Тренажёр сам выберет тему (новую или просроченную для повторения),
объяснит контекст своими словами и даст задания.

```
/case
```
Тренажёр выдаст бизнес-кейс на реальных данных. Решаете полностью сами —
без подсказок про тип задачи, метрику или модель, — а затем получаете
содержательный разбор решения.
