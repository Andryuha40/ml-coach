# /mlcoach — диспетчер веток тренажёра

Это точка входа. Логика каждой ветки — в отдельном файле рядом с этим
(`coach.md`, `case.md`, `interview.md`), намеренно не в этом файле — чтобы
не загружать в контекст все три режима на каждый вызов.

## Как выбрать ветку

1. **Если запрос явно называет ветку** — сразу прочитай нужный файл и
   следуй ему целиком, ничего из этого файла больше не нужно:
   - `/mlcoach coach`, `/coach`, "давай позанимаемся ML", "потренируй меня
     по теме X" → прочитай `coach.md`.
   - `/mlcoach case`, `/case`, "дай бизнес-кейс", "практика от постановки
     до решения" → прочитай `case.md`.
   - `/mlcoach interview`, `/interview`, "проведи собеседование", "хочу
     мок-интервью" → прочитай `interview.md`.

2. **Если неясно** — прежде чем читать файлы веток, поприветствуй
   пользователя одним сообщением и уточни детали. Не читай пока ни один
   из трёх файлов веток — только `_coach/progress.json` (нужен для
   последнего вопроса ниже).

   Сначала коротко (2-4 предложения) объясни, что это за скилл и как им
   пользоваться дальше: `mlcoach` — персональный ML-тренажёр поверх
   материалов курса (лекции, семинары, репозиторий Евгения Паточенко) —
   не справочник, а тренировка с реальной проверкой кода и содержательным
   разбором, а не просто "верно/неверно". В следующий раз можно сразу
   писать `/coach`, `/case`, `/interview`, называть тему по имени или
   просить сменить уровень — без этого приветствия.

   Дальше — задай уточняющие вопросы **одним сообщением**, а не серией
   реплик подряд. Если доступен инструмент для структурированных вопросов
   с вариантами ответа (как в Claude Code) — используй его для всех
   вопросов сразу; если недоступен (например, в Cowork) — задай тот же
   список ясно оформленным текстом (нумерованный список с вариантами).
   Вопросы:

   1. **Режим** — точечная тренировка по теме (`coach`), сквозной
      бизнес-кейс (`case`) или мок-собеседование (`interview`)?
   2. **Уровень подготовки** (необязательно, можно пропустить/"не важно") —
      Junior / Middle / Senior / Pro? Влияет на сложность вопросов и
      заданий во всех трёх режимах, можно сменить позже в любой момент.
   3. Только если по ответу похоже, что нужен `coach` (для `case`/
      `interview` этот вопрос не задавай — там свой выбор темы/кейса
      внутри ветки): если в `progress.json` уже есть темы не в статусе
      `not_started` — **какую тему** брать: конкретную по названию,
      продолжить по прогрессу (следующая по очереди / просроченная для
      повторения), или начать курс заново с самой первой темы? Если
      прогресса ещё нет вообще (все темы `not_started`) — этот вопрос не
      нужен, курс и так начнётся с первой темы.

   После ответов прочитай соответствующий файл ветки и следуй ему целиком,
   учитывая уже полученные ответы — не переспрашивай то, что пользователь
   только что сказал (например, если он назвал тему или уровень здесь, шаг
   1 в `coach.md`/`case.md`/`interview.md` про чтение `skill_tier` и выбор
   темы просто применяет уже известный ответ, а не спрашивает заново).

## Общее для всех веток

- Состояние всех веток лежит в `_coach/` (`curriculum.yaml`, `cases.yaml`,
  `progress.json`, `case_log.json`, `interview_log.json`) — подробности в
  каждом из файлов веток.
- Учебные конспекты — в `content/notes/` (используются `coach` и
  `interview`); реальные датасеты для кода — в `content/ml_basics_course/`
  (git submodule, обязателен для `case`; для `coach`/`interview` не
  обязателен, если `content/notes/` уже собран — перед первым
  использованием `case` убедись, что submodule инициализирован:
  `git submodule update --init`).
- Необязательный уровень подготовки (`skill_tier` в `progress.json`:
  `junior`/`middle`/`senior`/`pro`) можно задать или сменить в любой
  момент, в любой ветке, даже посреди сессии — подробности калибровки
  описаны в конце каждого файла ветки.
