# /mlcoach — диспетчер веток тренажёра

Это точка входа. Логика каждой ветки — в отдельном файле рядом с этим
(`coach.md`, `case.md`, `interview.md`), намеренно не в этом файле — чтобы
не загружать в контекст все три режима на каждый вызов.

## Как выбрать ветку

1. **Если запрос явно называет ветку** — сразу прочитай нужный файл и
   следуй ему целиком, ничего из этого файла больше не нужно:
   - `/mlcoach coach`, `/coach`, "давай позанимаемся ML", "потренируй меня
     по теме X" → прочитай `coach.md`.
   - `/mlcoach case`, `/case`, "дай бизнес-кейс", "практика от постановки
     до решения" → прочитай `case.md`.
   - `/mlcoach interview`, `/interview`, "проведи собеседование", "хочу
     мок-интервью" → прочитай `interview.md`.

2. **Если неясно** — коротко спроси пользователя, не читая пока ни один из
   трёх файлов:

   > Что нужно: точечная тренировка по теме (`coach`), сквозной бизнес-кейс
   > (`case`) или мок-собеседование (`interview`)?

   После ответа — прочитай соответствующий файл и следуй ему.

## Общее для всех веток

- Состояние всех веток лежит в `_coach/` (`curriculum.yaml`, `cases.yaml`,
  `progress.json`, `case_log.json`, `interview_log.json`) — подробности в
  каждом из файлов веток.
- Учебные конспекты — в `content/notes/` (используются `coach` и
  `interview`); реальные датасеты для кода — в `content/ml_basics_course/`
  (git submodule, обязателен для `case`; для `coach`/`interview` не
  обязателен, если `content/notes/` уже собран — перед первым
  использованием `case` убедись, что submodule инициализирован:
  `git submodule update --init`).
- Необязательный уровень подготовки (`skill_tier` в `progress.json`:
  `junior`/`middle`/`senior`/`pro`) можно задать или сменить в любой
  момент, в любой ветке, даже посреди сессии — подробности калибровки
  описаны в конце каждого файла ветки.
